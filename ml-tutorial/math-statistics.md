# 统计学：从数据中学习的科学

> **核心问题**：你只有有限的数据，如何推断整个世界的规律？你的推断有多可靠？

---

## 什么是统计学？——从零开始

### 一个生活中的故事

你在一家奶茶店打工。老板问你：「咱们店每天大概能卖多少杯？」你翻了翻过去 7 天的记录：23、18、31、25、20、28、22 杯。

你脑子里快速闪过几个念头：
- 「平均下来大概 24 杯左右吧」
- 「但每天波动挺大的，大概差个 5 杯上下」
- 「周一和周三似乎卖得特别多，是不是有什么规律？」

你刚刚做的——计算均值、衡量波动、寻找规律——这就是**统计学**的雏形。

### 直观理解

如果说**概率论**是「已知规则，预测结果」（比如已知硬币公平，抛出 10 次正面的概率是多少），那么**统计学**恰恰相反——「已知结果，推断规则」（比如抛了 100 次得到 60 次正面，这枚硬币公平吗？）。

统计学是一门**从数据中逆向推断**的学科。给你的永远是有限的、有噪声的样本，你要回答的永远是关于整个世界的结论。这个「逆向」的动作，让统计学充满了权衡、陷阱和艺术。

### 形式化定义

**统计学**是收集、整理、分析、解释数据，并从中得出关于总体结论的科学。它由两大分支构成：

| 分支 | 核心任务 | 典型问题 |
|------|---------|----------|
| **描述性统计** | 用几个数字概括数据 | 「这批数据的中心在哪？离散程度如何？」 |
| **推断性统计** | 从样本推断总体 | 「样本均值 24 杯，真正的日均销量是多少？」 |

### 为什么机器学习必须学统计学？

机器学习本质上是**大规模自动化的统计推断**。训练模型 = 用训练集（样本）估计模型参数（总体参数）。评估模型 = 判断在测试集上的表现是否能泛化到真实世界（从样本到总体的推断）。没有统计学，你就不知道——

- 你的模型真的学到了规律，还是只是在背训练集？
- 模型 A 比模型 B 高了 0.3% 的准确率——这是真实提升还是随机波动？
- 1000 个特征中选了 10 个——你确定它们不是碰巧相关？

统计学的每一项工具，都在回答你训练模型时必须面对的这些问题。

## 1. 总体与样本

### 生活例子

你想知道全中国成年人的平均身高。理想情况下，你应该测量**每一个人**（约 14 亿人），然后算出均值——这就是**总体（Population）**的真实均值 $\mu$。

现实中你做不到。你只能在某个城市的商场门口测量 500 个人——这 500 人就是你的**样本（Sample）**。

现在问题来了：如果这 500 个人恰好是在篮球赛散场时测量的，你的「样本均值 = 188cm」能代表全国人民吗？显然不能。这就是**抽样偏差**——统计推断灾难的根源。

### 直观理解

想象一个装满红球和白球的大桶（总体）。你伸手进去捞了一把（样本）。你只能看到捞出来的球，但你要猜整个桶里红球的比例。如果红球沉底而你只捞表面——你的推断就全错了。

**核心矛盾**：总体太大/太贵/不可能全量观测 → 只能用样本 → 样本有随机性 → 你的推断永远有不确定性。统计学就是一套**量化这种不确定性**的工具。

### 形式化定义

| 概念 | 记号 | 含义 | 举例 |
|------|------|------|------|
| 总体 | $N$ 个个体 | 研究对象的全体 | 全中国成年人的身高 |
| 总体参数 | 希腊字母（$\mu$, $\sigma$, $\pi$） | 总体的固定常数，未知 | $\mu = 167.3$（真实但不可知） |
| 样本 | $n$ 个个体（$n \ll N$） | 从总体中抽取的子集 | 测量的 500 个人 |
| 统计量 | 拉丁字母（$\bar{x}$, $s$, $p$） | 样本的函数，随机变量 | $\bar{x} = 170.1$（可算，但每次样本不同） |
| 简单随机抽样 | 每个个体等概率被选，独立 | 最基本的理想抽样方式 | 从身份证号中随机抽取 |

> 记号约定是统计学的「命名法」：**希腊字母 = 总体的未知常数；拉丁字母 = 样本的已知变量**。记住这个规则，你就能瞬间分辨一个符号在说什么。

### 手算示例

假设全国成年人的（假装已知的）身高数据——真实总体 $\mu = 167.3$, $\sigma = 8.5$。现在你手上有 5 个人的样本测量值：

$$x = \{172, 158, 165, 181, 169\}$$

手算样本统计量：

$$\bar{x} = \frac{172 + 158 + 165 + 181 + 169}{5} = \frac{845}{5} = 169.0$$

$$s^2 = \frac{(172-169)^2 + (158-169)^2 + (165-169)^2 + (181-169)^2 + (169-169)^2}{5-1}$$

$$= \frac{3^2 + (-11)^2 + (-4)^2 + 12^2 + 0^2}{4} = \frac{9 + 121 + 16 + 144 + 0}{4} = \frac{290}{4} = 72.5$$

$$s = \sqrt{72.5} \approx 8.51$$

样本均值 $169.0$ 不等于真实总体均值 $167.3$——这就是抽样误差。正常现象。

### Python 验证

```python
import numpy as np

np.random.seed(42)
population = np.random.normal(167.3, 8.5, 1_000_000)

sample = np.array([172, 158, 165, 181, 169])

xbar = sample.mean()
s2 = sample.var(ddof=1)
s = sample.std(ddof=1)

print(f"总体均值 μ = {population.mean():.1f} （真实值，现实中你永远不知道）")
print(f"样本均值 x̄ = {xbar:.1f}")
print(f"样本方差 s² = {s2:.1f}, 样本标准差 s = {s:.2f}")
print(f"误差 = {xbar - population.mean():.1f} → 抽样误差，n=5 时很常见")
```

### 常见误区

- **误区 1**：「样本够大就一定有代表性」——错。如果抽样方式有偏（如只测篮球场的人），样本再大也没用。**抽样方式比样本量更重要**。
- **误区 2**：「总体参数和样本统计量是一回事」——错。$\mu$ 是固定的常数，$\bar{x}$ 是随机变量（每次抽样可能不同）。

### ML 应用

训练集就是你的样本，测试集也是。你永远在**样本**上训练、在**样本**上评估——但期望模型在**总体**（真实世界未见数据）上表现好。这就是为什么你需要交叉验证、留出验证集、甚至部署后的在线监控——你永远在对抗从有限样本推断总体规律的不确定性。

## 2. 描述性统计

### 生活例子

你刚到一个新城市，打开租房 App，看到 100 套房子的月租金列表。面对 100 个数字，你怎么快速了解行情？你会忍不住想知道：
- 「大概多少钱？」（中心在哪）
- 「贵和便宜差多少？」（波动多大）
- 「是大多数人集中在某个价位，还是有特别多豪宅拉高均价？」（分布形状）

描述性统计就是帮你用几个数字「一句话描述」整批数据的工具。

### 2.1 集中趋势：数据的「中心」在哪

#### 直观理解

**均值**是数据的「重心」。想象数轴上每个数据点按权重 1 分布——均值就是平衡点。一个极端值会把平衡点拉向自己，就像跷跷板上突然坐了个大象。

**中位数**是数据的「正中间」。把所有数从小到大排列，正中间那个就是中位数。它不理睬极端值——大象坐上来，跷跷板中间位置不变。

**众数**是出现次数最多的值。对房租数据意义不大（连续值很少重复），但对「用户评分（1-5星）」这类离散数据很有用。

#### 手算示例

给出 9 个数据点：$\{3, 7, 5, 4, 6, 8, 2, 100, 5\}$

**均值**：
$$\bar{x} = \frac{3+7+5+4+6+8+2+100+5}{9} = \frac{140}{9} = 15.56$$

**中位数**：先排序 $\{2, 3, 4, 5, 5, 6, 7, 8, 100\}$，第 5 个 = 5

**众数**：5（出现了 2 次）

注意到什么？均值 15.56 被 100 这个极端值严重拉高，但中位数 5 基本不受影响。**当均值 >> 中位数，你就在看右偏分布**——少数极端大值拉高了均值。

#### Python 验证

```python
data = np.array([3, 7, 5, 4, 6, 8, 2, 100, 5])

print(f"均值:   {data.mean():.2f}")
print(f"中位数: {np.median(data):.1f}")
print(f"众数:   5（出现 {(data == 5).sum()} 次）")
print(f"均值({data.mean():.0f}) >> 中位数({np.median(data):.0f}) → 右偏分布，极端值在拉高均值")
```

### 2.2 离散程度：数据「散」得有多开

#### 直观理解

两批数据可以有相同的均值，但性质完全不同：
- A 班成绩：$\{60, 60, 60, 60, 60\}$，均值 60
- B 班成绩：$\{100, 20, 80, 40, 60\}$，均值 60

B 班的成绩明显比 A 班「散」。你需要一个数字来量化这个「散」。

**方差**：每个数据点离均值有多远的平方平均。用平方是为了不让正负偏差互相抵消。

**标准差**：方差的平方根——把它拉回原始单位。$\sigma$ 大 = 数据散得开。

**IQR（四分位距）**：Q3（75%分位数）- Q1（25%分位数），中间 50% 数据的范围。箱线图中「箱子」的高度。

#### 手算示例

用 B 班成绩 $\{100, 20, 80, 40, 60\}$ 手算方差和标准差：

均值 $\bar{x} = 60$

$$s^2 = \frac{(100-60)^2 + (20-60)^2 + (80-60)^2 + (40-60)^2 + (60-60)^2}{5-1}$$

$$= \frac{40^2 + (-40)^2 + 20^2 + (-20)^2 + 0^2}{4} = \frac{1600 + 1600 + 400 + 400 + 0}{4} = \frac{4000}{4} = 1000$$

$$s = \sqrt{1000} \approx 31.62$$

Q1 = 40（排序后 $\{20, 40, 60, 80, 100\}$，第 25% 位置），Q3 = 80，IQR = 80 - 40 = 40

#### Python 验证

```python
grades_b = np.array([100, 20, 80, 40, 60])

s2_manual = ((grades_b - grades_b.mean())**2).sum() / (len(grades_b) - 1)
s_manual = np.sqrt(s2_manual)
iqr_manual = np.percentile(grades_b, 75) - np.percentile(grades_b, 25)

print(f"方差 s² = {s2_manual:.0f}（手算：{1000}）")
print(f"标准差 s = {s_manual:.2f}（手算：{31.62}）")
print(f"IQR = {iqr_manual:.0f}（手算：{40}）")
```

### 2.3 偏度与峰度：分布的「形状」

#### 直观理解

**偏度**告诉你分布是不是歪的。「尾巴在哪边，就叫哪边偏」：
- **右偏（正偏）**：长尾拖在右边 → 少数极端大值 → 均值 > 中位数（收入分布最典型）
- **左偏（负偏）**：长尾拖在左边 → 少数极端小值 → 均值 < 中位数（考试简单时的高分分布）
- **对称**：偏度 ≈ 0 → 均值 ≈ 中位数

**峰度**告诉你尾部的「厚度」——极端值多不多：
- **高峰度**（> 3 或 excess > 0）→ 厚尾 → 极端事件比你预期的频繁（股票收益率就是厚尾——金融危机比正态分布预测的频繁得多）
- **低峰度**（< 3 或 excess < 0）→ 薄尾 → 像一个被压扁的钟
- **正态分布的峰度**精确为 3（excess kurtosis = 0）

#### 手算示例

用 6 个数据点 $\{2, 3, 5, 7, 9, 50\}$ 感受偏度：

均值 $\bar{x} = 12.67$，中位数 = 6

偏度公式：
$$g_1 = \frac{\frac{1}{n}\sum(x_i - \bar{x})^3}{\left[\frac{1}{n}\sum(x_i - \bar{x})^2\right]^{3/2}}$$

先算各离差：
- $(2-12.67)^3 = (-10.67)^3 = -1214.3$
- $(3-12.67)^3 = (-9.67)^3 = -904.2$
- $(5-12.67)^3 = (-7.67)^3 = -451.2$
- $(7-12.67)^3 = (-5.67)^3 = -182.3$
- $(9-12.67)^3 = (-3.67)^3 = -49.4$
- $(50-12.67)^3 = (37.33)^3 = 52019.3$

$\sum(x-\bar{x})^3 = -1214.3 - 904.2 - 451.2 - 182.3 - 49.4 + 52019.3 = 49217.9$

$\frac{1}{n}\sum(x-\bar{x})^3 = 49217.9 / 6 = 8203.0$

$\sum(x-\bar{x})^2$：$113.8+93.4+58.8+32.1+13.4+1393.8 = 1705.3$，$\frac{1}{n}\sum = 284.2$

$[\frac{1}{n}\sum(x-\bar{x})^2]^{3/2} = 284.2^{1.5} = 4791.4$

$$g_1 = \frac{8203.0}{4791.4} \approx 1.71$$

正值 → **右偏**。均值 12.67 >> 中位数 6，定性一致。

#### Python 验证

```python
from scipy import stats

data_skew = np.array([2, 3, 5, 7, 9, 50])

print(f"均值:   {data_skew.mean():.2f}")
print(f"中位数: {np.median(data_skew):.1f}")
print(f"偏度:   {stats.skew(data_skew, bias=False):.3f}")
print(f"峰度:   {stats.kurtosis(data_skew, bias=False):.3f}  (excess, 正态=0)")
print(f"→ 均值 >> 中位数，偏度 {stats.skew(data_skew, bias=False):.2f} > 0 → 右偏分布")
```

### 常见误区

- **误区**：「偏度正负看均值和中位数谁大」——这在大方向上是正确的经验法则，但不是严格成立。对称分布均值 = 中位数 = 众数。但偏度 > 0 不保证均值一定 > 中位数，只是大多数情况下成立。
- **误区**：「峰度就是『尖不尖』」——不准确。峰度主要描述**尾部**的厚度，而非峰顶的尖度。厚尾分布即使峰顶平坦，峰度也可以很高。

### ML 应用

- **偏度与特征工程**：收入、点击量等长尾特征需要 log 变换（$\log(x+1)$）来减轻右偏，帮助模型学习
- **峰度与异常值**：高峰度意味着你该检查异常值——这些极端点会严重影响基于 MSE（均方误差）的模型
- **描述性统计是模型的第一步**：在跑任何模型之前，先 `.describe()` 看一眼。均值接近中位数吗？标准差合理吗？min/max 有没有离谱值？这 30 秒的检查能帮你避免「用错误数据训练正确模型」的悲剧

## 3. 抽样分布

### 生活例子

你在一家餐厅当品控。今天的汤咸不咸，你尝一勺就够了——但你能保证这一勺代表整锅汤吗？如果你分别从锅的左、中、右各尝一勺，三勺的咸度肯定不完全相同。这三勺的咸度分布——就是「抽样分布」的直观写照。

### 直观理解

假设你知道全国成年人的真实身高分布（$\mu = 167.3$, $\sigma = 8.5$）。现在你做 1000 次「抽取 30 人的样本，计算样本均值」的实验。这 1000 个样本均值会形成一个分布——**抽样分布（Sampling Distribution）**。

抽样分布的核心发现：

1. **抽样分布的均值** ≈ 总体均值（样本均值是无偏的）
2. **抽样分布的标准差** = $\frac{\sigma}{\sqrt{n}}$，称为**标准误（Standard Error, SE）**
3. **$n$ 越大，抽样分布越窄**——大样本的估计更精确
4. **不管你原始数据长什么样，样本均值的抽样分布趋近正态**（中心极限定理）

### 形式化定义

$$\text{SE}(\bar{x}) = \frac{\sigma}{\sqrt{n}}$$

- $\sigma$ = 总体标准差
- $n$ = 样本量
- 直觉：$n$ 增大 100 倍 → SE 缩小 10 倍。精度提升的「性价比」递减（需要平方倍的样本量才能得到线性倍的精度提升）

### 手算演示

假设总体 $\sigma = 10$：

| 样本量 n | SE | 含义 |
|---------|-----|------|
| 4 | $10/2 = 5.0$ | 四分之一宽 |
| 25 | $10/5 = 2.0$ | 五分之一宽 |
| 100 | $10/10 = 1.0$ | 十分之一宽 |
| 400 | $10/20 = 0.5$ | 二十分之一宽 |

从 SE=5 降到 SE=0.5，你需要样本量从 4 增到 400——100 倍样本只换来 10 倍精度。

### Python 模拟

```python
np.random.seed(42)
population = np.random.randn(1_000_000) * 10 + 167.3

def simulate_se(n, n_trials=2000):
    means = np.array([np.random.choice(population, n).mean() for _ in range(n_trials)])
    return means

for n in [9, 25, 100, 400]:
    means = simulate_se(n)
    theoretical_se = 10 / np.sqrt(n)
    print(f"n={n:4d}: 模拟 SE={means.std():.3f}, 理论 SE={theoretical_se:.3f}, 均值均值={means.mean():.3f}")
```

### 常见误区

- **误区**：「标准差和标准误是一回事」——错。标准差描述**单个数据点**的波动（$\sigma \approx 10$ 表示个体身高差 10cm 很正常）；标准误描述**样本均值**的波动（SE=1 表示你用样本估计总体均值的误差约 1cm）。二者不是一回事，但通过 $n$ 连接。
- **误区**：「中心极限定理说数据本身会趋近正态」——错。CLT 说的是**样本均值的分布**趋近正态，跟原始数据长什么样没关系。你从指数分布抽样，样本均值仍然趋近正态。

### ML 应用

- **交叉验证的分数波动** = 抽样分布的体现。5 折 CV 的 5 个分数的标准差其实就是 SE 的估计
- **模型评估的样本量**：如果你在 100 个测试样本上得到准确率 90%，SE ≈ $\sqrt{0.9 \times 0.1 / 100} = 3\%$——你的真实准确率大概在 90% ± 6% 之间。样本太小时这个区间太宽，无法得出有意义的结论
- **早停（Early Stopping）**：验证集损失的波动本质上也是抽样误差，这就是为什么需要 patience 参数——别被一次波动骗了

## 4. 点估计与最大似然估计（MLE）

### 生活例子

你捡到一枚奇怪的硬币，你想知道它正面朝上的概率 $p$ 是多少。你抛了 10 次，得到：正、正、反、正、正、正、反、正、反、正——7 次正面。

你的直觉告诉你：$\hat{p} = 7/10 = 0.7$。这个直觉背后的数学原理，就是最大似然估计。

### 直观理解

MLE 的核心思想出奇地简单：**选一个参数值，使得「观察到当前数据」这件事的概率最大**。

具体来说：
- 如果真实 $p=0.1$，看到 7 次正面的概率是 $\binom{10}{7} \times 0.1^7 \times 0.9^3 \approx 0.000009$——几乎不可能
- 如果真实 $p=0.7$，看到 7 次正面的概率是 $\binom{10}{7} \times 0.7^7 \times 0.3^3 \approx 0.267$——挺大
- 如果真实 $p=1.0$，看到 7 次正面的概率是 0——不可能（因为已经出现了 3 次反面）

所以 $p=0.7$ 是让这个观察「最可能发生」的参数值。这就是 MLE。

### 4.1 Bernoulli 分布的 MLE——完整手算推导

#### 问题设定

你有 $n$ 次独立抛掷结果 $\{x_1, x_2, \ldots, x_n\}$，每个 $x_i \in \{0, 1\}$，1 表示正面。设 $P(X=1) = p$。求 $p$ 的 MLE。

#### 手算推导

**第 1 步**：写出似然函数——给定 $p$，观察到这个样本的概率：

$$L(p) = \prod_{i=1}^{n} p^{x_i}(1-p)^{1-x_i}$$

如果一个数据点是正面（$x_i=1$），贡献因子 $p$；如果是反面（$x_i=0$），贡献因子 $1-p$。

**第 2 步**：因为有 $k$ 个正面和 $n-k$ 个反面，似然函数简化为：

$$L(p) = p^{k}(1-p)^{n-k}$$

其中 $k = \sum_{i=1}^{n} x_i$。

**第 3 步**：取对数（乘积变求和，计算更方便——但不改变最大值的位置）：

$$\ell(p) = \ln L(p) = k \ln p + (n-k) \ln(1-p)$$

**第 4 步**：对 $p$ 求导并令其为零：

$$\frac{d\ell}{dp} = \frac{k}{p} - \frac{n-k}{1-p} = 0$$

$$\frac{k}{p} = \frac{n-k}{1-p}$$

$$k(1-p) = (n-k)p$$

$$k - kp = np - kp$$

$$k = np$$

**第 5 步**：解出 $\hat{p}_{\text{MLE}}$：

$$\boxed{\hat{p}_{\text{MLE}} = \frac{k}{n}}$$

**结论**：正面次数除以总次数——最直觉的答案就是你最可能想到的答案。

#### 手算验证

10 次抛掷，7 次正面：$\hat{p} = 7/10 = 0.7$

| $p$ | 似然 $L(p) = p^7(1-p)^3$ |
|-----|--------------------------|
| 0.3 | $0.3^7 \times 0.7^3 = 0.000219 \times 0.343 = 0.000075$ |
| 0.5 | $0.5^7 \times 0.5^3 = 0.0078125 \times 0.125 = 0.000977$ |
| 0.7 | $0.7^7 \times 0.3^3 = 0.082354 \times 0.027 = 0.002224$ |
| 0.9 | $0.9^7 \times 0.1^3 = 0.478297 \times 0.001 = 0.000478$ |

**$p=0.7$ 的似然最大** → MLE 估计量 $\hat{p} = 0.7$ 正确。

### 4.2 Gaussian 分布的 MLE——完整手算推导

#### 问题设定

你的样本 $\{x_1, \ldots, x_n\}$ 来自正态分布 $N(\mu, \sigma^2)$，$\mu$ 和 $\sigma^2$ 都未知。求两者的 MLE。

#### 手算推导

**第 1 步**：正态分布的概率密度函数：

$$f(x \mid \mu, \sigma^2) = \frac{1}{\sqrt{2\pi\sigma^2}} \exp\!\left(-\frac{(x-\mu)^2}{2\sigma^2}\right)$$

**第 2 步**：似然函数（独立观测的联合密度）：

$$L(\mu, \sigma^2) = \prod_{i=1}^{n} \frac{1}{\sqrt{2\pi\sigma^2}} \exp\!\left(-\frac{(x_i-\mu)^2}{2\sigma^2}\right)$$

$$= (2\pi\sigma^2)^{-n/2} \exp\!\left(-\frac{1}{2\sigma^2}\sum_{i=1}^n (x_i - \mu)^2\right)$$

**第 3 步**：对数似然：

$$\ell(\mu, \sigma^2) = -\frac{n}{2}\ln(2\pi) - \frac{n}{2}\ln\sigma^2 - \frac{1}{2\sigma^2}\sum_{i=1}^{n}(x_i - \mu)^2$$

**第 4 步**：对 $\mu$ 求偏导并令其为零：

$$\frac{\partial \ell}{\partial \mu} = \frac{1}{\sigma^2}\sum_{i=1}^{n}(x_i - \mu) = 0$$

$$\boxed{\hat{\mu}_{\text{MLE}} = \frac{1}{n}\sum_{i=1}^{n} x_i = \bar{x}}$$

**第 5 步**：对 $\sigma^2$ 求偏导并令其为零（将 $\mu = \bar{x}$ 代入后）：

$$\frac{\partial \ell}{\partial \sigma^2} = -\frac{n}{2\sigma^2} + \frac{1}{2\sigma^4}\sum_{i=1}^{n}(x_i - \bar{x})^2 = 0$$

两边乘以 $2\sigma^2$：$-n + \frac{1}{\sigma^2}\sum(x_i - \bar{x})^2 = 0$

$$\boxed{\hat{\sigma}^2_{\text{MLE}} = \frac{1}{n}\sum_{i=1}^{n}(x_i - \bar{x})^2}$$

**注意**：分母是 $n$，不是 $n-1$。这个 $\hat{\sigma}^2_{\text{MLE}}$ 是**有偏的**——它系统性地低估了真实方差。

#### 手算验证

用 5 个数 $\{2, 4, 6, 8, 10\}$ 手算：

$\hat{\mu} = \frac{2+4+6+8+10}{5} = \frac{30}{5} = 6$

$\hat{\sigma}^2 = \frac{(2-6)^2 + (4-6)^2 + (6-6)^2 + (8-6)^2 + (10-6)^2}{5}$
$= \frac{16 + 4 + 0 + 4 + 16}{5} = \frac{40}{5} = 8$

如果真实 $\sigma^2 = 10$，MLE 给出了 8——系统性地偏小了。

### Python 验证

```python
from scipy.optimize import minimize

np.random.seed(42)
true_mu, true_sigma = 5.0, 2.0
data = np.random.normal(true_mu, true_sigma, 100)

# 解析解
mu_mle = data.mean()
sigma2_mle = ((data - mu_mle) ** 2).mean()
sigma_mle = np.sqrt(sigma2_mle)

print(f"真实 μ = {true_mu},       MLE μ̂ = {mu_mle:.4f}")
print(f"真实 σ² = {true_sigma**2},   MLE σ̂² = {sigma2_mle:.4f}")
print(f"真实 σ = {true_sigma},       MLE σ̂ = {sigma_mle:.4f}")

# 数值优化验证
def neg_log_likelihood(params, data):
    mu, log_sigma = params
    sigma = np.exp(log_sigma)
    n = len(data)
    return n * np.log(sigma) + 0.5 * np.sum(((data - mu) / sigma) ** 2)

result = minimize(neg_log_likelihood, x0=[0, 0], args=(data,), method='L-BFGS-B')
mu_num, sigma_num = result.x[0], np.exp(result.x[1])
print(f"\n数值优化: μ̂ = {mu_num:.4f}, σ̂ = {sigma_num:.4f}")
```

### 常见误区

- **误区 1**：「MLE 总是最优的」——MLE 在样本量大时有很好的性质（一致、渐近正态、渐近有效），但小样本时可能表现很差（如 $\hat{\sigma}^2_{\text{MLE}}$ 有偏）。而且遇到复杂模型，MLE 很容易过拟合。
- **误区 2**：「似然函数就是概率密度」——在离散情况下似然确实是概率；但在连续情况下，似然是概率**密度**的乘积，不是概率（某个特定值的概率对连续分布永远是 0）。所以 MLE 最大化的是「密度乘积」，不是真正的「概率」。

### ML 应用

| ML 模型 | 概率假设 | MLE ↔ 损失函数 |
|---------|----------|----------------|
| 线性回归 | $y \sim N(f(x), \sigma^2)$ | MLE ↔ MSE（均方误差） |
| 逻辑回归 | $y \sim \text{Bernoulli}(p(x))$ | MLE ↔ 交叉熵损失 |
| Softmax 分类 | $y \sim \text{Categorical}(p_1,\ldots,p_K)$ | MLE ↔ 多类交叉熵 |
| Poisson 回归 | $y \sim \text{Poisson}(\lambda(x))$ | MLE ↔ Poisson 损失 |

**理解 MLE = 理解深度学习中几乎所有损失函数的来源**。

## 5. 最大后验估计（MAP）

### 生活例子

你从来没见过这个人，他连续投了 3 次篮球，全中。他的命中率是多少？

- MLE 告诉你：$\hat{p} = 3/3 = 100\%$——但你觉得可能吗？一个人投篮百发百中？
- 你的常识告诉你：大多数业余玩家命中率大概 50% 左右，投 10 次也就中 5-6 个。3 次全中只是「今天运气好」。

你大脑自动融入了「先验知识」——这就是 MAP 的思想。

### 直观理解

MLE 只看数据，完全不关心参数本身是否「合理」。MAP 在 MLE 上加入**先验分布**——在没看到数据之前，你认为参数大概率应该是什么值？

MAP = MLE × 先验。数学上：

$$\hat{\theta}_{\text{MAP}} = \mathop{\arg\max}_{\theta} \underbrace{P(\theta \mid X)}_{\text{后验}} = \mathop{\arg\max}_{\theta} \underbrace{P(X \mid \theta)}_{\text{似然}} \cdot \underbrace{P(\theta)}_{\text{先验}}$$

取对数：

$$\hat{\theta}_{\text{MAP}} = \mathop{\arg\max}_{\theta} \left[ \underbrace{\ell(\theta)}_{\text{对数似然}} + \underbrace{\ln P(\theta)}_{\text{对数先验}} \right]$$

### MAP 与正则化的联系

这是 ML 中最重要的联系之一：

| 先验分布 | 对数先验项 | 对应 | 效果 |
|----------|-----------|------|------|
| 高斯 $N(0, \tau^2)$ | $-\frac{\theta^2}{2\tau^2}$ | **L2（Ridge）** | 参数缩小但不为 0 |
| 拉普拉斯 $Lap(0, b)$ | $-\frac{|\theta|}{b}$ | **L1（Lasso）** | 部分参数精确归 0 |
| 均匀先验 | 0 | **MLE** | 无正则化 |

**L1 为什么产生稀疏解？** 拉普拉斯分布在 0 点有一个尖峰——对数先验在 0 点的惩罚「最陡」，把不重要的参数直接推到 0。

### 手算示例：硬币 + Beta 先验

3 次投篮全中。用 Beta 先验来编码「我认为大概是 50% 左右」：

- 先验：$\text{Beta}(\alpha=5, \beta=5)$——均值 0.5，程度适中的信念
- 数据：3 次成功，0 次失败

**Beta-Binomial 共轭**：后验也是 Beta 分布：

$$\text{后验} = \text{Beta}(\alpha + k, \; \beta + n - k) = \text{Beta}(5+3, \; 5+0) = \text{Beta}(8, 5)$$

MAP 估计量（后验众数）：
$$\hat{p}_{\text{MAP}} = \frac{\alpha + k - 1}{\alpha + \beta + n - 2} = \frac{5 + 3 - 1}{5 + 5 + 3 - 2} = \frac{7}{11} \approx 0.636$$

对比：MLE = 3/3 = 1.0，MAP = 0.636——先验把极端估计「拽」回了合理范围。

### Python 验证

```python
from scipy.stats import beta as beta_dist

heads, tails = 3, 0

priors = {
    "无信息 Beta(1,1)": (1, 1),
    "弱先验 Beta(2,2)": (2, 2),
    "中等先验 Beta(5,5)": (5, 5),
    "强先验 Beta(50,50)": (50, 50),
}

print(f"数据: {heads} 次成功, {tails} 次失败")
print(f"MLE: {heads/(heads+tails):.3f}\n")

for name, (a, b) in priors.items():
    a_post, b_post = a + heads, b + tails
    map_est = (a_post - 1) / (a_post + b_post - 2)
    post_mean = a_post / (a_post + b_post)
    ci_lo = beta_dist.ppf(0.025, a_post, b_post)
    ci_hi = beta_dist.ppf(0.975, a_post, b_post)
    print(f"{name}:")
    print(f"  MAP = {map_est:.3f}, 后验均值 = {post_mean:.3f}")
    print(f"  95% 可信区间: [{ci_lo:.3f}, {ci_hi:.3f}]")
```

### 常见误区

- **误区**：「MAP 比 MLE 更好」——取决于你的先验是否合理。如果先验严重偏离真实情况，MAP 会被先验「绑架」，在大量数据面前才慢慢纠正。
- **误区**：「MAP 和 MLE 在大样本下一样」——没错（似然项随 $n$ 增大而主导），但前提是先验不是「病态的」（如不允许参数取某些值）。

### ML 应用

- **L1 / L2 正则化** = MAP with Laplace / Gaussian prior——你现在知道为什么 Ridge 把参数缩小、Lasso 把参数归零了
- **贝叶斯神经网络**：给每个权重一个先验分布，训练后得到权重的后验分布——可以量化预测的不确定性
- **冷启动问题**：没有用户行为数据时，用先验（如「大多数人喜欢热门商品」）做推荐

## 6. 估计量的性质

### 直观理解

你面前有三个人，都在猜这个教室的温度：
- 甲说 25°C（但你知道他的温度计坏了，总是高 5 度）
- 乙说 20°C（但他的温度计有时偏高有时偏低，波动很大）
- 丙说 21°C（温度计很稳，而且长期来看平均就是真实值）

你更相信谁？我们需要一套标准来判断估计量的好坏。

### 形式化定义

| 性质 | 含义 | 数学表述 | 比喻 |
|------|------|---------|------|
| **无偏性** | 估计量的期望 = 真实值 | $\mathbb{E}[\hat{\theta}] = \theta$ | 丙的体温计长期不偏 |
| **一致性** | 样本越大越准 | $\hat{\theta} \xrightarrow{p} \theta$ 当 $n \to \infty$ | 多测几次，总归会逼近真值 |
| **有效性** | 在无偏估计量中方差最小 | $\text{Var}[\hat{\theta}]$ 最小 | 乙的体温计波动小，读数稳 |

### 手算验证：为什么 /(n-1) 而不是 /n？

这是统计学最经典的困惑。用 3 个数 $\{1, 4, 7\}$ 来演示。

已知真实均值 $\mu = 4$（如果这 3 个数就是总体的话）。总体的方差（如果用 $/n$）：

$$\sigma^2 = \frac{(1-4)^2 + (4-4)^2 + (7-4)^2}{3} = \frac{9+0+9}{3} = 6$$

现在假设你**不知道** $\mu=4$，只能从这 3 个数据点估计方差。

**关键来了**：$\bar{x}$ 是专门为了「让偏差平方和最小」而计算出来的——用 $\bar{x}$ 算出来的偏差平方和 $\sum(x_i - \bar{x})^2$ 一定 ≤ 用真实 $\mu$ 算出来的 $\sum(x_i - \mu)^2$！

所以 MLE 方差系统性地低估了真实方差。需要做一个校正：

**自由度** = 独立信息的数量。$n$ 个数据点中有一个「被消耗」去估计均值了，剩下只有 $n-1$ 个真正独立的信息用于估计方差。除以 $n-1$ 补偿了这个损失。

### Python 模拟验证

```python
np.random.seed(42)
true_var = 25
n_small = 5
n_sim = 100000

mle_vars, unb_vars = [], []
for _ in range(n_sim):
    sample = np.random.normal(0, np.sqrt(true_var), n_small)
    mu_hat = sample.mean()
    mle_vars.append(((sample - mu_hat) ** 2).sum() / n_small)
    unb_vars.append(((sample - mu_hat) ** 2).sum() / (n_small - 1))

print(f"真实方差: {true_var}")
print(f"MLE 方差均值 (/n): {np.mean(mle_vars):.2f} → 偏小，有偏")
print(f"无偏方差均值 (/(n-1)): {np.mean(unb_vars):.2f} → 无偏 ✓")
```

### 常见误区

- **误区**：「无偏估计就一定是好的」——错。一个疯狂的无偏估计：随机选一个 $x_i$ 作为 $\mu$ 的估计——它无偏（$\mathbb{E}[x_i] = \mu$），但方差极大，完全没用。实际中我们更愿意接受轻微偏差换取大幅方差降低（正则化就是这种权衡的体现）。
- **误区**：「除以 n-1 是因为这样方差更大」——因果搞反了。除以 n-1 不是因为要「变大」，而是因为自由度少了 1。

### ML 应用

- **偏差-方差分解**：ML 的核心理论「泛化误差 = 偏差² + 方差 + 不可约噪声」直接发源于估计理论
- **正则化**就是故意引入偏差来降低方差——你对 MAP 和 Ridge/Lasso 的理解现在闭合了
- **集成学习**：Bagging 降低方差，Boosting 降低偏差——每个算法家族的侧重点根源在于估计量性质

## 7. 置信区间

### 生活例子

你是一家药厂的质检员。你抽取了 30 片药，测得平均有效成分含量 $\bar{x} = 100$mg。你能直接交报告写「含量 = 100mg」吗？不能——你心里清楚，再抽 30 片很可能得到 98mg 或 103mg。

你想说的是：「我有 95% 的把握，真实含量在 95mg 到 105mg 之间。」——这就是**置信区间**。

### 直观理解

置信区间传达的信息不是「真值在哪里」，而是**「基于当前样本，我的估计有多不确定」**。

**关键纠正（极其重要）**：如果你算出了一个 95% 置信区间 $[95, 105]$，**不能**说「真实参数有 95% 的概率落在这个区间内」。在频率学派框架下，参数是固定常数不存在概率。正确解释是：

> **「如果我把『抽样 → 算区间』这个流程独立重复 100 次，大约 95 次算出来的区间会包含真实参数。」**

贝叶斯学派的「95% 可信区间」才支持「参数有 95% 概率落在此区间」的直觉解释。

### 手算示例：均值的 95% 置信区间

你有 10 个数据点：$\{72, 68, 75, 70, 73, 67, 71, 69, 74, 66\}$

**第 1 步**：计算样本均值和标准差
$$\bar{x} = \frac{72+68+75+70+73+67+71+69+74+66}{10} = \frac{705}{10} = 70.5$$

各离差平方：$(1.5)^2 + (-2.5)^2 + (4.5)^2 + (-0.5)^2 + (2.5)^2 + (-3.5)^2 + (0.5)^2 + (-1.5)^2 + (3.5)^2 + (-4.5)^2$
$= 2.25 + 6.25 + 20.25 + 0.25 + 6.25 + 12.25 + 0.25 + 2.25 + 12.25 + 20.25 = 82.5$

$$s = \sqrt{\frac{82.5}{9}} = \sqrt{9.167} \approx 3.028$$

**第 2 步**：计算标准误
$$\text{SE} = \frac{s}{\sqrt{n}} = \frac{3.028}{\sqrt{10}} = \frac{3.028}{3.162} \approx 0.957$$

**第 3 步**：查 t 分布临界值（$df = n-1 = 9$，95% 双侧）

$t_{0.025, 9} \approx 2.262$（查表）

**第 4 步**：构造置信区间
$$CI = \bar{x} \pm t_{\alpha/2, df} \times SE = 70.5 \pm 2.262 \times 0.957$$
$$= 70.5 \pm 2.165 = [68.33, \; 72.67]$$

### Python 验证与模拟

```python
from scipy.stats import t

data = np.array([72, 68, 75, 70, 73, 67, 71, 69, 74, 66])
n = len(data)
xbar = data.mean()
se = data.std(ddof=1) / np.sqrt(n)
t_crit = t.ppf(0.975, df=n-1)
ci = (xbar - t_crit * se, xbar + t_crit * se)
print(f"95% CI: [{ci[0]:.2f}, {ci[1]:.2f}]")
print(f"手算: [{68.33}, {72.67}] → 一致 ✓")

# 模拟验证覆盖率
n_sim = 1000; captured = 0; true_mu = 70.0
for _ in range(n_sim):
    sim_data = np.random.normal(true_mu, 3, n)
    xb = sim_data.mean(); s2 = sim_data.var(ddof=1)
    lo = xb - t.ppf(0.975, df=n-1) * np.sqrt(s2/n)
    hi = xb + t.ppf(0.975, df=n-1) * np.sqrt(s2/n)
    if lo <= true_mu <= hi: captured += 1
print(f"覆盖率: {captured/n_sim*100:.1f}%（应接近 95%）")
```

### 常见误区

- **误区 1**：「95% CI = 真实值有 95% 概率在里面」——频率学派不支持此解释（见上）。
- **误区 2**：「CI 越窄越好，样本越多越好」——窄 CI 的前提是抽样方式正确。如果抽样有偏，再窄的 CI 也偏离真值。
- **误区 3**：「CI 区间内的值都一样好」——CI 中间的点比两端的点更可能接近真值（从似然函数形状看），但频率学派不做此区分。

### ML 应用

- **模型评估**：报告「测试准确率 92.3%（95% CI: [90.1%, 94.5%]）」比单独报告 92.3% 诚实得多
- **A/B 测试**：报告「实验组转化率高 1.2%（95% CI: [0.3%, 2.1%]）」——CI 不跨越 0，你才知道提升是统计显著的

## 8. 假设检验

### 生活例子

你怀疑楼下超市的秤不准。你买了标注「500g」的苹果 10 袋，回家称了一下：503、498、502、497、505、501、499、504、496、500g。平均 500.5g。你觉得这秤不准吗？还是这 0.5g 只是正常误差？

假设检验用严格的数学回答：「**这 0.5g 的差异，是真实的（秤确实不准），还是随机波动就能解释的？**」

### 直观理解

假设检验的思维框架像法庭审判：

| 统计概念 | 法庭类比 |
|----------|---------|
| **原假设 $H_0$** | 被告无罪（假定没差异，待证据推翻） |
| **备择假设 $H_1$** | 被告有罪（你试图证明的差异） |
| **p 值** | 「假设被告确实无罪，但碰巧出现了这么不利的证据」的概率 |
| **显著性水平 $\alpha$** | 判案标准（$\alpha=0.05$ → 误判无罪的最高容忍率 5%） |
| **拒绝 $H_0$** | 判有罪 |
| **不拒绝 $H_0$** | 证据不足，不能判有罪（≠ 证明无罪） |
| **I 类错误** | 冤枉好人（误拒 $H_0$，假阳性） |
| **II 类错误** | 放走真凶（漏过 $H_1$，假阴性） |

### 手算 p 值：Binomial 精确检验

你抛硬币 20 次，15 次正面。这枚硬币公平吗？

$H_0: p = 0.5$，$H_1: p \neq 0.5$，$\alpha = 0.05$

在 $p=0.5$ 下，20 次中恰好 $k$ 次正面的概率：
$$P(X = k) = \binom{20}{k} \times 0.5^{20}$$

| k | $\binom{20}{k}$ | $P(X=k)$ |
|---|----------------|----------|
| 15 | 15504 | 0.01479 |
| 16 | 4845 | 0.00462 |
| 17 | 1140 | 0.00109 |
| 18 | 190 | 0.00018 |
| 19 | 20 | 0.00002 |
| 20 | 1 | 0.00000 |

右侧极端概率：$P(X \ge 15) = 0.01479 + 0.00462 + 0.00109 + 0.00018 + 0.00002 \approx 0.02069$

因为是**双侧检验**（检验的是「不公平」，不关心偏大还是偏小），p 值 ≈ 2 × 单侧概率：

$$p \approx 2 \times 0.02069 \approx 0.0414$$

$p = 0.0414 < 0.05$ → 在 $\alpha = 0.05$ 水平下**拒绝 $H_0$**，这枚硬币很可能不公平。

### 手算 p 值：t 检验（苹果重量）

$H_0: \mu = 500$，$H_1: \mu \neq 500$。$\alpha = 0.05$。

数据：n=10，$\bar{x}=500.5$。各离差平方和：
$3^2+(-2)^2+2^2+(-3)^2+5^2+1^2+(-1)^2+4^2+(-4)^2+0^2 = 9+4+4+9+25+1+1+16+16+0 = 85$

$s = \sqrt{85/9} = \sqrt{9.444} \approx 3.073$

$$t = \frac{\bar{x} - \mu_0}{s / \sqrt{n}} = \frac{500.5 - 500}{3.073 / \sqrt{10}} = \frac{0.5}{0.972} \approx 0.514$$

df=9，双侧。|t|=0.514 的 p 值远超 0.05 → **不拒绝 $H_0$**。0.5g 的差异完全可能是随机波动。

### Python 验证

```python
from scipy.stats import binomtest, ttest_1samp

result_binom = binomtest(15, 20, p=0.5, alternative='two-sided')
print(f"Binomial 检验: p = {result_binom.pvalue:.4f}（手算 ≈ 0.0414）")

apples = np.array([503, 498, 502, 497, 505, 501, 499, 504, 496, 500])
t_stat, p_val = ttest_1samp(apples, 500)
print(f"t 检验: t = {t_stat:.3f}, p = {p_val:.4f}（手算 t ≈ 0.514）")
```

### 常见误区

- **误区 1（最常见）**：「p 值是 $H_0$ 为真的概率」——**大错**。p 值是在 $H_0$ 为真这个**前提**下，观察到当前数据（或更极端数据）的概率。
- **误区 2**：「p > 0.05 说明没有差异」——只能说明「证据不足以推翻 $H_0$」，可能是样本量不够（功效不足）。
- **误区 3**：「p < 0.05 就是有意义的发现」——p 值告诉我们差异是否「真实」，但**不告诉我们差异有多大**（需要效应量）。

### ML 应用

- **特征选择**：用 t 检验、卡方检验筛选与标签显著相关的特征
- **A/B 测试**：判断新模型是否显著优于旧模型
- **模型比较**：判断模型 A 的 AUC 提升是真实的还是运气

## 9. 常见检验方法

### 9.1 t 检验

| 类型 | 场景 | scipy 函数 |
|------|------|-----------|
| 单样本 t | 「这批产品的平均重量是 500g 吗？」 | `ttest_1samp` |
| 独立双样本 t | 「A 组和 B 组的成绩是否不同？」 | `ttest_ind` |
| 配对 t | 「同一批人减肥前后体重是否有变化？」 | `ttest_rel` |

#### 手算独立双样本 t 检验

A 组：$\{5, 7, 6, 8, 9\}$，B 组：$\{2, 4, 3, 5, 4\}$

$\bar{x}_A = 7.0$, $\bar{x}_B = 3.6$, $s_A^2 = 2.5$, $s_B^2 = 1.3$, $n_A = n_B = 5$

合并方差：$s_p^2 = \frac{4 \times 2.5 + 4 \times 1.3}{8} = \frac{10 + 5.2}{8} = 1.9$

$$t = \frac{7.0 - 3.6}{\sqrt{1.9 \times 0.4}} = \frac{3.4}{\sqrt{0.76}} = \frac{3.4}{0.872} \approx 3.90$$

df=8，|t|=3.90 → p ≈ 0.0046 → **显著**。

```python
from scipy.stats import ttest_ind
A = np.array([5, 7, 6, 8, 9]); B = np.array([2, 4, 3, 5, 4])
t_stat, p_val = ttest_ind(A, B)
print(f"独立双样本 t: t = {t_stat:.3f}, p = {p_val:.6f}（手算 t ≈ 3.90 ✓）")
```

### 9.2 卡方检验（$\chi^2$ 检验）

#### 直观理解

「性别」和「是否购买某商品」有没有关联？

| | 购买 | 未购买 |
|---|------|--------|
| 男 | 40 | 60 |
| 女 | 30 | 70 |

如果无关，期望频数 = 行合计 × 列合计 / 总计：

| | 购买 | 未购买 |
|---|------|--------|
| 男 | 100×70/200=35 | 100×130/200=65 |
| 女 | 100×70/200=35 | 100×130/200=65 |

$$\chi^2 = \frac{(40-35)^2}{35} + \frac{(60-65)^2}{65} + \frac{(30-35)^2}{35} + \frac{(70-65)^2}{65}$$

$$= \frac{25}{35} + \frac{25}{65} + \frac{25}{35} + \frac{25}{65} = 0.714 + 0.385 + 0.714 + 0.385 = 2.198$$

df = (2-1)×(2-1) = 1。$\chi^2 = 2.198$, df=1 → p ≈ 0.138 > 0.05 → **不显著**。

```python
from scipy.stats import chi2_contingency
observed = np.array([[40, 60], [30, 70]])
chi2, p_val, dof, _ = chi2_contingency(observed)
print(f"卡方检验: χ² = {chi2:.3f}, p = {p_val:.3f}（手算 χ² ≈ 2.198 ✓）")
```

## 10. Bootstrap

### 生活例子

你手上有 30 个苹果的重量。你算了中位数 = 152g。但你没法再去苹果园采更多苹果了——你只有这 30 个。请问你算出来的中位数 152g 靠谱吗？可能的误差范围是多大？

传统方法需要你假设数据服从正态分布，用公式推导标准误。但如果数据不服从正态分布（比如有很多极端大苹果），这个推导就不可靠。有没有不依赖任何分布假设的方法？

### 直观理解

Bootstrap 的想法异常简单，也异常强大：

> **既然我只有这一份样本，那就假装这份样本就是「总体」，从里面反复有放回地抽样。**

具体步骤：
1. 从原始样本（n=30）中有放回地抽 30 个（每次可能重复抽到同一个人）
2. 计算这个重抽样本的中位数
3. 重复步骤 1-2 共 10000 次
4. 这 10000 个重抽中位数形成的分布 = 中位数的经验抽样分布
5. 取第 2.5 百分位和第 97.5 百分位 → 95% Bootstrap 置信区间

### 手算示例（简化）

原始数据（n=6）：$\{12, 15, 18, 22, 25, 30\}$，中位数 = 20

Bootstrap 重抽样 5 次（实际应做 1000+ 次，这里仅演示）：

| 轮次 | 重抽样结果 | 中位数 |
|------|-----------|--------|
| 1 | $\{22, 12, 30, 12, 25, 15\}$ | 18.5 |
| 2 | $\{15, 18, 25, 15, 30, 18\}$ | 18.0 |
| 3 | $\{12, 22, 22, 30, 30, 25\}$ | 23.5 |
| 4 | $\{18, 12, 12, 25, 18, 22\}$ | 18.0 |
| 5 | $\{25, 25, 15, 18, 18, 12\}$ | 18.0 |

Bootstrap 中位数在 18~23.5 波动 → 你可以估计 SE 和 CI。

### Python 实现

```python
np.random.seed(42)
data = np.random.exponential(scale=10, size=40)  # 非正态数据

def bootstrap_ci(data, statistic, B=10000, alpha=0.05):
    n = len(data)
    stats = np.empty(B)
    for i in range(B):
        resample = np.random.choice(data, size=n, replace=True)
        stats[i] = statistic(resample)
    lo = np.percentile(stats, 100 * alpha / 2)
    hi = np.percentile(stats, 100 * (1 - alpha / 2))
    return lo, hi, stats

ci_lo, ci_hi, boot_stats = bootstrap_ci(data, np.median, B=10000)
print(f"样本中位数: {np.median(data):.2f}")
print(f"Bootstrap SE(median): {boot_stats.std():.3f}")
print(f"95% Bootstrap CI: [{ci_lo:.2f}, {ci_hi:.2f}]")
```

### 常见误区

- **误区 1**：「Bootstrap 能创造新信息」——不能。Bootstrap 只能告诉你**已有样本的不确定性**。如果原始样本本身有偏，Bootstrap 得出的所有结论都有偏。
- **误区 2**：「Bootstrap 对任何事情都有效」——不。对极端的统计量（如 min, max），Bootstrap 表现很差。

### ML 应用

- **Bagging**——随机森林的核心。每个决策树训练在一个 Bootstrap 样本上
- **模型指标的不确定性**——用 Bootstrap 计算 AUC、F1 等任何指标的置信区间
- **Out-of-bag (OOB) 评估**——随机森林中约 37% 的样本没被抽到，可作为免费验证集

## 11. 相关性 ≠ 因果性

### 生活例子

数据表明：
- 冰淇淋销量和溺水死亡人数高度正相关
- 一个国家的巧克力消费量和诺贝尔奖得主数量正相关
- 尼古拉斯·凯奇每年拍的电影数和游泳池溺亡人数相关

禁止冰淇淋能减少溺水吗？多吃巧克力能得诺贝尔奖吗？让凯奇少拍片能挽救生命吗？**当然不能。但数据确实给出了高相关性！**

### 直观理解

两个变量 $X$ 和 $Y$ 相关可能因为：

```
X → Y     直接因果（吸烟 → 肺癌）
Y → X     反向因果（有跑步习惯 → 身体好 → 更喜欢户外活动）
Z → X, Y  混杂变量（夏天高温 → 冰淇淋↑ + 溺水↑）
巧合       伪相关（数据量足够大时，总会有些变量碰巧相关）
```

**相关性是统计概念**（数据和数据之间的关系），**因果性是物理概念**（一个事件确实导致了另一个事件）。

### 形式化：Simpson 悖论

| | 治疗 A | 治疗 B |
|---|--------|--------|
| 轻症 | 95/100 (95%) | 20/25 (80%) |
| 重症 | 50/100 (50%) | 200/350 (57%) |
| **总计** | 145/200 (72.5%) | 220/375 (58.7%) |

分组看，治疗 A 在轻症和重症中都优于治疗 B！但汇总后 A 看起来更差——因为 A 大部分是重症患者（治愈率低），B 大部分是轻症患者。**分组变量的分布不均导致汇总数据扭曲了真实效果**。

### Python 演示

```python
np.random.seed(42)
temp = np.random.normal(25, 5, 200)
ice_cream = 50 + 10 * temp + np.random.randn(200) * 15
drowning = 2 + 0.5 * temp + np.random.randn(200) * 3

corr_ic_d = np.corrcoef(ice_cream, drowning)[0, 1]
print(f"冰淇淋 vs 溺水 相关系数: {corr_ic_d:.3f}（高度正相关！）")

# 偏相关：控制温度后
from scipy.stats import pearsonr
resid_ic = ice_cream - np.polyval(np.polyfit(temp, ice_cream, 1), temp)
resid_dr = drowning - np.polyval(np.polyfit(temp, drowning, 1), temp)
partial_corr, _ = pearsonr(resid_ic, resid_dr)
print(f"偏相关（控制温度后）: {partial_corr:.3f} → 归零：消除混杂后两者无关")
```

### 常见误区

- **误区**：「特征重要性高 = 因果效应强」——SHAP 值、permutation importance 衡量的是**预测贡献**，不是因果效应。一个虚假相关但高预测力的特征（如「用户 ID 最后一位」）可以特征重要性很高，但没有因果关系。
- **误区**：「控制所有变量就能消除混杂」——还可能引入对撞偏差（Collider Bias）、过度控制等问题。

### ML 应用

- 特征重要性 ≠ 因果关系——不要基于 SHAP 值做重大决策
- 因果推断（Double ML, Causal Forest, DoWhy）是 ML 的前沿方向
- 推荐系统需区分「推荐起了作用（因果）」还是「用户本来就想买（关联）」

## 12. 效应量

### 生活例子

你做了一个 A/B 测试：新推荐系统让用户点击率从 5.00% 提升到 5.02%。p = 0.001（高度显著——因为你测了 100 万用户）。

你应该全面推广新系统吗？0.02% 的点击率提升在商业上可能**完全无关紧要**。p 值告诉你的只是「差异不太可能是随机引起的」，但**没有告诉你差异有多大**。

### 直观理解

**p 值受样本量驱动**——样本量足够大时，哪怕差异微乎其微，p 值也可以极小。效应量则**不依赖样本量**，它直接衡量差异的实际大小。

### Cohen's d

$$d = \frac{\bar{x}_1 - \bar{x}_2}{s_{\text{pooled}}}$$

$$s_{\text{pooled}} = \sqrt{\frac{(n_1-1)s_1^2 + (n_2-1)s_2^2}{n_1 + n_2 - 2}}$$

| d | 解释 | 两组重叠度 |
|---|------|-----------|
| 0.2 | 小差异 | 约 85% |
| 0.5 | 中等差异 | 约 67% |
| 0.8 | 大差异 | 约 53% |

### 手算示例

A 组（n=8）：$\{3,5,4,6,5,7,4,6\}$ → $\bar{x}_A=5.0$, $s_A=1.309$
B 组（n=8）：$\{8,10,9,7,11,8,9,10\}$ → $\bar{x}_B=9.0$, $s_B=1.309$

$$s_{\text{pooled}} = \sqrt{\frac{7 \times 1.309^2 + 7 \times 1.309^2}{14}} = 1.309$$

$$d = \frac{9.0 - 5.0}{1.309} = 3.06$$——**巨大**的效应量。

### Python 实现

```python
def cohens_d(g1, g2):
    n1, n2 = len(g1), len(g2)
    var1, var2 = g1.var(ddof=1), g2.var(ddof=1)
    pooled = np.sqrt(((n1-1)*var1 + (n2-1)*var2) / (n1+n2-2))
    return (g2.mean() - g1.mean()) / pooled

# 大样本微差异 → p 显著但 d 很小
g1 = np.random.normal(100, 15, 5000)
g2 = np.random.normal(100.3, 15, 5000)
t_s, p_s = ttest_ind(g1, g2); d_s = cohens_d(g1, g2)
print(f"大样本微差异: p={p_s:.4f}, d={d_s:.4f} → p 可能显著但效应量极小")

# 小样本大差异 → p 不显著但 d 很大
g3 = np.random.normal(100, 10, 12)
g4 = np.random.normal(115, 10, 12)
t_s2, p_s2 = ttest_ind(g3, g4); d_s2 = cohens_d(g3, g4)
print(f"小样本大差异: p={p_s2:.4f}, d={d_s2:.3f} → 效应量很大但 p 不显著")
```

### 常见误区

- **误区**：「显著就是重要」——**这是统计应用中最大的误区之一**。p 值回答「差异是否真实」，效应量回答「差异是否重要」。两个都应该报告。

### ML 应用

- 模型对比报告中同时给出 p 值和效应量
- 不要只报告「模型 A 显著优于 B（p < 0.05）」，还要报告「F1 提升了 2.3 个百分点，Cohen's d = 0.42」

## 13. 统计功效与样本量

### 生活例子

你用 20 个用户测试新旧方案对比。新方案转化率高 5%，但 p = 0.23——不显著。你老板说：「新方案不行，放弃。」

但你心里嘀咕：5% 的提升在商业上很大，p 不显著可能只是因为**样本量太小**，没有足够的「火力」检测到这个差异。就像在漆黑的大厅里用手电筒找一只小黑猫——你可能已经把光打到了猫身上，但手电筒太暗，看不清。

**功效（Power）**就是你确实能看到那只猫的能力——「如果差异真实存在，我的实验有多大概率能检测到它？」

### 直观理解

| 概念 | 含义 | 通常取值 |
|------|------|---------|
| $\alpha$（I 类错误率） | 假阳性——无差异却声称有 | 0.05 |
| $\beta$（II 类错误率） | 假阴性——有差异却没发现 | ≤ 0.2 |
| **功效 $1-\beta$** | 有差异时正确检出的概率 | ≥ 0.8 (80%) |
| 效应量 | 差异的实际大小 | Cohen's d 等 |

### 手算：所需样本量（双样本 t 检验，每组等量）

$$n \approx \frac{2(z_{\alpha/2} + z_{\beta})^2}{d^2}$$

$z_{0.025}=1.96$, $z_{0.2}=0.842$

- 对 $d=0.5$：$n \approx \frac{2(1.96+0.842)^2}{0.25} \approx 63$ 人/组
- 对 $d=0.2$：$n \approx \frac{2(1.96+0.842)^2}{0.04} \approx 393$ 人/组

**效应量从 0.5 降到 0.2，所需样本量暴涨 6 倍以上**——检测小差异极其昂贵。

### Python 实现

```python
from scipy.stats import norm

def required_n_per_group(d, alpha=0.05, power=0.8):
    z_alpha = norm.ppf(1 - alpha / 2)
    z_beta = norm.ppf(power)
    return int(np.ceil(2 * (z_alpha + z_beta)**2 / d**2))

for d in [0.2, 0.5, 0.8]:
    n = required_n_per_group(d)
    print(f"d={d}: n={n}/组, 总计至少 {2*n} 人")

# 模拟验证
n_sim, d_true = 64, 0.5
rejections = sum(1 for _ in range(5000)
    if ttest_ind(np.random.normal(0,1,n_sim), np.random.normal(d_true,1,n_sim))[1] < 0.05)
print(f"模拟功效（n=64/组, d=0.5）: {rejections/5000:.3f}（理论 ≈ 0.80）")
```

### 常见误区

- **误区**：「不显著就是没有差异」——这可能是 **II 类错误**！你的实验可能功效不足，漏掉了真实差异。永远检查功效再下结论。
- **误区**：「先收数据再看 p 值」——这叫 p-hacking / 数据窥探。应在实验**开始前**就基于功效分析确定样本量。
- **误区**：「功效 0.8 就够了」——功效 0.8 意味着即使真实差异存在，你也有 20% 的概率（1/5！）漏掉它。

### ML 应用

- **A/B 测试设计**：确定需要多少用户参与测试才有足够检测能力
- **模型评估的样本量规划**：采集标注数据前，估算需要多少测试样本才能可靠比较模型
- **早停的 patience**：本质上在控制「把随机波动误判为模型退化」的风险

## 14. 多重检验校正

### 生活例子

你在超市里随机尝了 20 种不同品牌的饼干，发现其中一个牌子「明显比其他好吃」。你兴奋地告诉朋友：「我发现最好吃的饼干了！」

但你想过吗——如果你随机尝 20 种，**纯碰运气**也可能遇到一种「看起来特别好」的。你尝的牌子越多，碰运气找到「显著好吃」的概率越高。这就是多重检验问题。

### 直观理解

每次检验 $\alpha = 0.05$ → 有 5% 概率假阳性。做 $m$ 次独立检验，至少出现一次假阳性的概率：

| m | FWER | m | FWER |
|---|------|---|------|
| 1 | 5% | 10 | 40% |
| 20 | 64% | 50 | 92% |
| 100 | 99.4% | 1000 | ≈ 100% |

公式：$\text{FWER} = 1 - (1 - 0.05)^m$

### 校正方法

**Bonferroni 校正**（最保守）：$\alpha_{\text{adjusted}} = \alpha / m$
- 优点：严格控制 FWER；缺点：太保守，几乎没有检验能通过。

**Benjamini-Hochberg (BH)**（最常用，控制 FDR）：
1. 把 p 值从小到大排序
2. 找到最大的 $k$ 使得 $p_{(k)} \leq \frac{k}{m} \cdot q$（$q$ 通常 0.05）
3. 拒绝前 $k$ 个

### 手算示例

5 个 p 值：$\{0.001, 0.012, 0.023, 0.045, 0.300\}$，BH 校正（$q=0.05$）：

| rank | p | 阈值 $rank/5 \times 0.05$ | 是否显著 |
|------|---|-------------------------|---------|
| 1 | 0.001 | 0.01 | ✓ |
| 2 | 0.012 | 0.02 | ✓ |
| 3 | 0.023 | 0.03 | ✓ |
| 4 | 0.045 | 0.04 | ✗ |
| 5 | 0.300 | 0.05 | ✗ |

k=3，拒绝前 3 个。第 4 个（0.045）被 BH 过滤。Bonferroni 阈值 = 0.05/5=0.01，只剩第 1 个。

### Python 实现

```python
np.random.seed(42)
m_tests = 1000; n_true_effect = 50
true_effects = np.zeros(m_tests); true_effects[:n_true_effect] = 0.5
p_values = np.array([ttest_1samp(np.random.normal(eff,1,30),0)[1] for eff in true_effects])

print(f"未校正显著数: {(p_values<0.05).sum()}")
tp = ((p_values<0.05) & (true_effects>0)).sum()
fp = ((p_values<0.05) & (true_effects==0)).sum()
print(f"  真阳性 {tp}, 假阳性 {fp}, FDR {fp/max((p_values<0.05).sum(),1):.3f}")

# BH 校正
def bh_correction(p_vals, q=0.05):
    m = len(p_vals); sorted_idx = np.argsort(p_vals)
    sorted_p = p_vals[sorted_idx]
    thresholds = np.arange(1, m+1)/m * q
    significant = sorted_p <= thresholds
    if not significant.any(): return np.zeros(m, dtype=bool)
    k_max = np.where(significant)[0].max()
    rejected = np.zeros(m, dtype=bool)
    rejected[sorted_idx[:k_max+1]] = True
    return rejected

bh_sig = bh_correction(p_values, q=0.05)
bh_tp = (bh_sig & (true_effects>0)).sum(); bh_fp = (bh_sig & (true_effects==0)).sum()
print(f"BH 校正: 显著 {bh_sig.sum()} 个, 真阳性 {bh_tp}, 假阳性 {bh_fp}, FDR {bh_fp/max(bh_sig.sum(),1):.3f}")
```

### 常见误区

- **误区**：「做了多重检验校正就万事大吉」——校正只能解决统计上的多重检验问题，不能解决数据质量问题（如选择性报告、p-hacking）。

### ML 应用

- **特征选择**：对 1000 个特征逐一做显著性检验 → **必须**多重检验校正
- **超参数搜索**：Grid Search 100 组超参数的最佳性能可能只是「碰巧」——需嵌套交叉验证评估

## 15. 非参数检验

### 生活例子

你分析网店的用户评分（1-5 星），想比较两种商品的好评度。但评分是**序数数据**——你不能说「4 星减 2 星 = 2，5 星减 3 星 = 2，所以这两对差距一样大」。这个减法没有意义（你不知道评分之间的"心理距离"是否相等）。

t 检验假设数据是连续的、等距的、近似正态的。评分数据不满足——你需要**非参数检验**。

### 直观理解

| 非参数检验 | 替代 | 核心思路 |
|-----------|------|---------|
| Mann-Whitney U | 独立 t 检验 | 混合排序，如果同分布则秩应均匀交错 |
| Wilcoxon 符号秩 | 配对 t 检验 | 算每对差值、排名、检验符号 |
| Kruskal-Wallis | 单因素 ANOVA | Mann-Whitney 推广到多组 |
| Spearman $\rho$ | Pearson r | 对秩算相关系数（检测单调关系） |

### 手算：Mann-Whitney U（简化）

A 组：$\{3, 8, 6\}$，B 组：$\{1, 2, 4\}$

混合排序：1(B), 2(B), 3(A), 4(B), 6(A), 8(A)
A 的秩和 $R_A = 3+5+6 = 14$

$$U_A = R_A - \frac{n_A(n_A+1)}{2} = 14 - \frac{3 \times 4}{2} = 8$$

$U_B = 3 \times 3 - 8 = 1$，$U = \min(8, 1) = 1$

$U=1$ 非常小 → A 组几乎全部排在 B 组之上 → 两组差异大 → p 小。

### Python 验证

```python
from scipy.stats import mannwhitneyu, spearmanr, shapiro

np.random.seed(42)
g1 = np.random.exponential(2, 30); g2 = np.random.exponential(3, 30)

_, p_n1 = shapiro(g1); _, p_n2 = shapiro(g2)
print(f"正态检验: g1 p={p_n1:.4f}, g2 p={p_n2:.4f} → 都不正态，用非参数")

u, p_mw = mannwhitneyu(g1, g2, alternative='two-sided')
_, p_t = ttest_ind(g1, g2)
print(f"Mann-Whitney p = {p_mw:.4f}, t 检验 p = {p_t:.4f}")

# Spearman vs Pearson
x = np.random.exponential(2, 50)
y = 2 * x + np.random.normal(0, 2, 50)
sp_r, _ = spearmanr(x, y)
pr_r = np.corrcoef(x, y)[0, 1]
print(f"Spearman ρ = {sp_r:.3f}, Pearson r = {pr_r:.3f}（Spearman 对异常值鲁棒）")
```

### 常见误区

- **误区**：「非参数检验在任何情况下都比参数检验好」——如果数据确实来自正态分布，参数检验的功效更高。非参数检验的代价是功效略低。
- **误区**：「数据看着不正态就该用非参数」——大样本时（>30）CLT 保证了 t 检验对正态性偏离有一定鲁棒性。

### ML 应用

- 处理长尾分布特征（收入、浏览量、交易金额）
- Spearman 相关常用于特征工程的关联分析
- 序数特征（如评分、等级）的显著性检验

## 16. ANOVA（方差分析）

### 生活例子

你想比较三种教学方法的效果。如果两两做 t 检验，需要做 3 次——多重检验问题！如果比较 10 种方法，需要做 45 次两两比较——大多数都可能是假阳性。

**ANOVA 用一个检验回答：「这些组的均值全都相等吗？」**如果 ANOVA 显著，再做事后两两比较（此时已校正了多重检验）。

### 直观理解

ANOVA 的核心直觉：**比较两种方差**。

$$F = \frac{\text{组间方差}}{\text{组内方差}}$$

- $F \approx 1$：组间差异 ≈ 组内随机波动 → 各组差不多
- $F \gg 1$：组间差异远大于组内随机波动 → 各组均值确实不同

### 手算完整 ANOVA 表

A: $\{70, 75, 80\}$, B: $\{80, 85, 90\}$, C: $\{60, 65, 70\}$

总均值 $\bar{x} = 75$, $\bar{x}_A=75$, $\bar{x}_B=85$, $\bar{x}_C=65$

**SS_between**：$3 \times 0 + 3 \times 100 + 3 \times 100 = 600$
**SS_within**：A 内部 50 + B 内部 50 + C 内部 50 = 150
**SS_total**：600 + 150 = 750

| 来源 | SS | df | MS | F |
|------|----|----|----|---|
| 组间 | 600 | 2 | 300 | 300/25=**12** |
| 组内 | 150 | 6 | 25 | |
| 总计 | 750 | 8 | | |

$F = 12.0$, df=(2,6)。查表 $F_{0.05,2,6} = 5.14$。$12.0 > 5.14$ → **拒绝 $H_0$**。

效应量 $\eta^2 = 600/750 = 0.80$——非常大，组间差异解释了 80% 的总变异。

### Python 验证

```python
from scipy.stats import f_oneway
from itertools import combinations

method_a = np.array([70,75,80]); method_b = np.array([80,85,90]); method_c = np.array([60,65,70])
f_stat, p_val = f_oneway(method_a, method_b, method_c)
print(f"ANOVA: F = {f_stat:.4f}, p = {p_val:.6f}（手算 F=12.0 ✓）")

all_data = np.concatenate([method_a, method_b, method_c])
ss_between = sum(len(g)*(g.mean()-all_data.mean())**2 for g in [method_a,method_b,method_c])
ss_total = ((all_data-all_data.mean())**2).sum()
print(f"η² = {ss_between/ss_total:.4f}（效应量 0.80 → 大）")

# Tukey HSD post-hoc
from scipy.stats import studentized_range
k, N = 3, len(all_data); groups = [method_a, method_b, method_c]
ms_within = sum(((g-g.mean())**2).sum() for g in groups) / (N-k)
q_crit = studentized_range.ppf(0.95, k, N-k)
print("\n=== Tukey HSD ===")
for (i,gi),(j,gj) in combinations(enumerate(groups),2):
    diff = abs(gi.mean()-gj.mean())
    se = np.sqrt(ms_within/2*(1/len(gi)+1/len(gj)))
    sig = "显著" if diff > q_crit*se else "不显著"
    print(f"  组{i} vs 组{j}: diff={diff:.1f} → {sig}")
```

### 常见误区

- **误区**：「ANOVA 显著 = 所有组之间都有差异」——只告诉你「至少有一组不同」，不告诉哪组。需要 post-hoc 检验。
- **误区**：「ANOVA 可以比较任何数据」——需要方差齐性。不满足时用 Welch's ANOVA 或 Kruskal-Wallis。

### ML 应用

- **特征选择核心**：`sklearn.feature_selection.f_classif` = ANOVA F 值
- **多组模型对比**：比较 3+ 种模型性能是否有显著差异
- **超参数影响分析**：判断超参数不同取值是否系统性地影响模型性能

## 17. 残差分析

### 生活例子

你开了一家面包店，根据过去 30 天的数据训练了一个模型预测明天的销量：
$$\text{预测销量} = 50 + 2 \times \text{温度}$$

模型看起来不错（$R^2 = 0.82$），但你怎么确定这个**线性**模型是合理的？也许销量和温度的关系其实是「天太热了大家不出门，销量反而下降」的倒 U 形？

**残差 = 真实值 - 预测值**。如果模型正确，残差应该是纯粹的随机噪声——没有模式、没有趋势、均匀随机地散布在 0 附近。如果残差有模式，说明模型漏掉了某些信息。

### 直观理解

| 诊断图 | 检查什么 | 好模型 | 坏信号 |
|--------|---------|--------|--------|
| 残差 vs 拟合值 | 线性性、同方差性 | 随机散布 | U 形→非线性；漏斗→异方差 |
| Q-Q 图 | 残差正态性 | 点在对角线上 | 两端偏离→厚尾；S 形→偏态 |
| Scale-Location | 同方差性 | 水平线 | 有趋势→异方差 |
| Cook's Distance | 高影响点 | $D_i < 4/n$ | $D_i > 4/n$ → 该点过度影响模型 |

### 手算演示（简化）

5 个点 $\{(1,2), (2,4), (3,6), (4,8), (5,18)\}$，拟合线 $\hat{y}=1.6x+0.8$

| x | y 真值 | y 预测 | 残差 |
|---|--------|--------|------|
| 1 | 2 | 2.4 | -0.4 |
| 2 | 4 | 4.0 | 0.0 |
| 3 | 6 | 5.6 | 0.4 |
| 4 | 8 | 7.2 | 0.8 |
| 5 | 18 | 8.8 | **9.2** ← 异常！ |

残差 9.2 暴露了问题——要么是异常值（录入错误），要么关系不是线性的。

### Python 完整诊断

```python
from scipy.stats import probplot
from numpy.linalg import inv

np.random.seed(42)
n = 80; X_raw = np.random.uniform(0, 10, n)
y = 3 * X_raw + 5 + np.random.normal(0, 3, n)

X_design = np.column_stack([np.ones(n), X_raw])
beta = inv(X_design.T @ X_design) @ X_design.T @ y
y_pred = X_design @ beta; residuals = y - y_pred
p = X_design.shape[1]; mse = np.sum(residuals**2) / (n - p)

H = X_design @ inv(X_design.T @ X_design) @ X_design.T
leverage = np.diag(H)
std_residuals = residuals / np.sqrt(mse * (1 - leverage))

print(f"残差均值: {residuals.mean():.6f} (应≈0)")

(theo_q, ordered_res), _ = probplot(std_residuals, dist="norm")
qq_corr = np.corrcoef(theo_q, ordered_res)[0, 1]
print(f"Q-Q 相关性: {qq_corr:.4f} (越接近1越正态)")

cooks_d = (std_residuals**2 * leverage) / (p * (1 - leverage))
influential = np.where(cooks_d > 4/n)[0]
print(f"高影响点 (D > 4/n): {len(influential)} 个")
if len(influential): print(f"  索引: {influential.tolist()}")

dw = np.sum(np.diff(residuals)**2) / np.sum(residuals**2)
print(f"Durbin-Watson: {dw:.4f} (≈2→无自相关)")
```

### 常见误区

- **误区**：「$R^2$ 高就可以不看残差了」——$R^2$ 只衡量整体拟合度，不揭示系统性偏差。$R^2 = 0.9$ 的模型可能仍有 U 形残差模式。
- **误区**：「残差正态就是好模型」——还需要无趋势（线性性）、方差均匀（同方差性）、无自相关。

### ML 应用

- 任何回归模型都该做残差分析
- 残差的模式 = 特征工程的线索（如残差随温度增大 → 加上温度²试试）
- 时间序列预测中的残差自相关检验

## 18. 频率学派 vs 贝叶斯学派

### 直观理解

统计学内部有两套哲学体系，对「概率」是什么有根本分歧：

**频率学派（t 检验、p 值、置信区间）**：
- 概率 = 长期频率。参数 $\mu$ 是固定未知常数。
- 「95% 置信区间」≠「$\mu$ 有 95% 概率在区间内」，而是「重复实验时 95% 的 CI 会包含 $\mu$」。

**贝叶斯学派（MAP、后验分布）**：
- 概率 = 信念程度。参数 $\mu$ 是随机变量，有分布。
- 「95% 可信区间」=「$\mu$ 有 95% 概率在区间内」——直觉友好的解释。

**一句话总结**：频率学派「参数固定、数据随机」；贝叶斯学派「数据固定（已看到）、参数随机」。

### Python 对比

```python
from scipy.stats import beta as beta_dist

heads, tails = 14, 6
# 频率学派
result = binomtest(heads, heads+tails, p=0.5, alternative='two-sided')
print(f"频率学派: MLE = {heads/(heads+tails):.3f}, p = {result.pvalue:.4f}")

# 贝叶斯学派（Beta(1,1) 无信息先验）
a_post, b_post = 1+heads, 1+tails
ci_lo = beta_dist.ppf(0.025, a_post, b_post)
ci_hi = beta_dist.ppf(0.975, a_post, b_post)
print(f"贝叶斯学派: 后验均值 = {a_post/(a_post+b_post):.3f}")
print(f"  95% 可信区间: [{ci_lo:.3f}, {ci_hi:.3f}] → p 有 95% 概率在此区间")
```

### ML 应用

- **频率学派方法**：MLE、交叉验证、A/B 测试、大多数 sklearn 模型
- **贝叶斯学派方法**：朴素贝叶斯、贝叶斯优化、贝叶斯神经网络、概率编程
- **实际中两者共存**：正则化（MAP）就是贝叶斯思想嵌入频率学派框架的产物

## 思考题（10 题）

### 题目 1：抽样分布

从均值为 100、标准差为 15 的总体中，你抽取了 n = 36 的样本。

(a) 样本均值的标准误是多少？
(b) 样本均值落在 [95, 105] 之间的概率大约是多少？
(c) 如果 n 变成 144，标准误变为多少？

**解答**：

(a) SE = $15 / \sqrt{36} = 15 / 6 = 2.5$

(b) 根据中心极限定理，$\bar{x} \sim N(100, 2.5^2)$。$z_1 = (95-100)/2.5 = -2$, $z_2 = (105-100)/2.5 = 2$。$P(-2 < Z < 2) \approx 0.9545$。所以大约 95.45%。

(c) SE = $15 / \sqrt{144} = 15 / 12 = 1.25$。样本量 ×4，精度（SE 的倒数）×2——精度提升"性价比"递减。

### 题目 2：MLE 推导 —— 指数分布

指数分布 $f(x|\lambda) = \lambda e^{-\lambda x}$（$x \ge 0$）。给定独立同分布样本 $\{x_1, \ldots, x_n\}$，求 $\lambda$ 的 MLE。

**解答**：

似然：$L(\lambda) = \prod_{i=1}^n \lambda e^{-\lambda x_i} = \lambda^n e^{-\lambda \sum x_i}$

对数似然：$\ell(\lambda) = n \ln \lambda - \lambda \sum_{i=1}^n x_i$

求导：$\frac{d\ell}{d\lambda} = \frac{n}{\lambda} - \sum x_i = 0$

$$\boxed{\hat{\lambda}_{\text{MLE}} = \frac{n}{\sum x_i} = \frac{1}{\bar{x}}}$$

直觉：指数分布的均值 = $1/\lambda$，所以 MLE 就是用样本均值替代总体均值反解出 $\lambda$。

### 题目 3：方差估计的偏差

为什么样本方差除以 $n-1$ 而不是 $n$？给出一个直观解释。

**解答**：

核心原因：你用 $\bar{x}$（样本均值）代替 $\mu$（总体均值）来计算离差平方和时，$\sum(x_i - \bar{x})^2$ 总是**小于或等于** $\sum(x_i - \mu)^2$——因为 $\bar{x}$ 的定义就是让这个平方和**最小**的点。

因此，分母用 $n$ 会导致系统性低估（有偏）。除以 $n-1$ 恰好校正这个偏差——$n-1$ 代表「自由度」：$n$ 个数据点中有一个被用于估计均值，剩下 $n-1$ 个独立信息用于估计方差。

验证：$\mathbb{E}[s^2_{n-1}] = \sigma^2$，而 $\mathbb{E}[s^2_{n}] = \frac{n-1}{n}\sigma^2 < \sigma^2$。

### 题目 4：置信区间的正确解释

你从 25 个样本算出 95% 置信区间为 [45, 55]。以下哪些说法是**正确**的？

(a) 真实均值有 95% 的概率落在 [45, 55] 内
(b) 如果重复实验 100 次，大约 95 个 CI 会包含真实均值
(c) 真实均值一定在 45 到 55 之间
(d) 有 5% 的概率真实均值不在这个区间内

**解答**：

(a) **错误**——频率学派框架下参数不是随机变量，不存在概率分布
(b) **正确**——这是频率学派 95% CI 的标准解释
(c) **错误**——不能保证 100% 包含，大约有 5% 的 CI 不包含
(d) **错误**——这是贝叶斯可信区间的解释，不是频率学派 CI 的解释。频率学派下不能说"有 5% 的概率不在区间内"，因为区间是随机的、参数是固定的

### 题目 5：p 值的解释

某研究 p = 0.03。以下说法中，哪些是**正确且完整**的？

(a) $H_0$ 为真的概率是 3%
(b) 如果 $H_0$ 为真，观察到当前或更极端结果的概率是 3%
(c) 有 97% 的概率 $H_1$ 为真
(d) 在 $\alpha = 0.05$ 下应拒绝 $H_0$

**解答**：

(a) **错误**——p 值不是 $P(H_0|\text{data})$，而是 $P(\text{data}|H_0)$。方向搞反了。
(b) **正确**——这是 p 值的准确定义。
(c) **错误**——p 值不能转化为 $H_1$ 为真的概率。这种转化需要贝叶斯框架（需要先验）。
(d) **正确**——p < $\alpha$，按预设规则应拒绝 $H_0$。

### 题目 6：效应量与样本量

研究者 A 报告：新药比安慰剂平均多降了 2 mmHg 血压，n=1000，p < 0.001。
研究者 B 报告：新药比安慰剂平均多降了 8 mmHg 血压，n=60，p = 0.08。

谁的发现更有实际意义？为什么？

**解答**：

研究者 B 的发现更有实际意义。虽然 p 值不显著，但效应量很大（8 mmHg vs 仅 2 mmHg）。2 mmHg 即使统计显著，临床意义有限——这么小的差异几乎不影响患者的健康结局。而 8 mmHg 是临床上**有意义**的差异（根据高血压治疗指南，降压 5-10 mmHg 可显著降低心血管事件风险）。

研究者 B 的 p = 0.08 很可能是因为样本太小（功效不足），不是因为没有差异。B 不应该轻易放弃，而是需要更大样本的验证实验。

**核心教训**：p 值告诉你"差异是否真实"，效应量告诉你"差异是否重要"。永远两个一起报告。

### 题目 7：多重检验

你在 200 个特征上分别做了 t 检验，未校正时 12 个特征显著（p < 0.05）。

(a) 用 Bonferroni 校正后，哪些假设被拒绝？
(b) 用 BH 校正（q=0.05），12 个 p 值从小到大为：0.0001, 0.001, 0.003, 0.007, 0.012, 0.018, 0.023, 0.030, 0.035, 0.040, 0.045, 0.049。哪些被拒绝？

**解答**：

(a) Bonferroni: $\alpha_{\text{adj}} = 0.05/200 = 0.00025$。只有 p < 0.00025 的被拒绝。12 个显著中只有第 1 个（0.0001）通过 Bonferroni。太保守了！

(b) BH: 计算每个排名的阈值 $\frac{k}{200} \times 0.05$：

| k | p 值 | 阈值 k/200×0.05 | 是否显著 |
|---|------|----------------|---------|
| 1 | 0.0001 | 0.00025 | ✓ |
| 2 | 0.001 | 0.0005 | ✗ |
| 3 | 0.003 | 0.00075 | ✗ |
| ... | ... | ... | ... |
| 12 | 0.049 | 0.003 | ✗ |

实际上只有第 1 个（p=0.0001）通过 BH 校正（$0.0001 < 0.00025$）！第 2 个的阈值是 0.0005 但 p=0.001 > 0.0005，所以到 k=1 就断了。

注意：即使 BH 也只拒绝了 1 个——不是 BH 保守，而是 200 个检验中只有一个非常极端的 p 值。这恰恰说明未校正时那 12 个"显著"中，大多数可能是假阳性。

### 题目 8：Bootstrap

你只有 5 个数据点 $\{3, 7, 2, 9, 5\}$，想估计中位数的标准误。不能用公式（非正态分布）。

(a) 描述 Bootstrap 步骤
(b) 用 Python 实现

**解答**：

(a) Bootstrap 步骤：
1. 从 5 个数据点中有放回地抽 5 个（允许重复）
2. 计算重抽样本的中位数
3. 重复 B=10000 次
4. 取 10000 个中位数的标准差 → 中位数的 Bootstrap SE
5. 取第 2.5 和 97.5 百分位 → 95% Bootstrap CI

(b) Python 实现：

```python
data = np.array([3, 7, 2, 9, 5])
B = 10000
np.random.seed(42)
boot_medians = np.array([
    np.median(np.random.choice(data, size=5, replace=True))
    for _ in range(B)
])
print(f"样本中位数: {np.median(data)}")
print(f"Bootstrap SE: {boot_medians.std():.3f}")
print(f"95% Bootstrap CI: [{np.percentile(boot_medians, 2.5):.1f}, {np.percentile(boot_medians, 97.5):.1f}]")
```

注意：n=5 非常小，Bootstrap 的准确性取决于原始样本的代表性。如果这 5 个点有偏，Bootstrap 结果也会有偏。

### 题目 9：相关性 vs 因果性

给出一个「两个变量高度相关但不是因果关系」的真实例子，并解释可能的混杂因素。

**解答**：

**例：某城市「消防员出动数量」和「火灾造成的财产损失」高度正相关（r ≈ 0.85）。**

这不是因果关系——出动更多消防员**并不会导致**更多财产损失。**混杂变量是「火灾的严重程度」**：

- 大火 → 出动更多消防员（消防站的合理反应）
- 大火 → 更大财产损失

所以严重程度同时导致了更多的消防员出动和更大的财产损失。

如果**控制**严重程度（只看同等严重程度的火灾），两者之间的相关性就消失了——甚至可能**逆转**：在同等严重的火灾中，出动更多消防员可能意味着更快扑灭，从而**减少**财产损失。

这个例子完美展示了为什么不能仅凭相关性推断因果关系。

### 题目 10：ANOVA 效应量

单因素 ANOVA，3 组（每组 n=20），结果如下：
- SS_between = 1800
- SS_within = 5700
- SS_total = 7500

(a) 计算 F 统计量和 $\eta^2$
(b) 判断效应量大小
(c) 如果 ANOVA 显著，下一步应该做什么？

**解答**：

(a) k = 3, N = 60
- df1 = k-1 = 2, df2 = N-k = 57
- MS_between = 1800/2 = 900
- MS_within = 5700/57 = 100
- F = 900/100 = **9.0**
- $\eta^2 = 1800/7500 = 0.24$

(b) 根据 Cohen 的经验标准：$\eta^2$ ≈ 0.01（小）, 0.06（中）, 0.14（大）。0.24 > 0.14 → **大效应量**。分组变量（如教学方法）解释了 24% 的总变异——这是非常有实际意义的差异。

(c) 由于 ANOVA 显著（F=9.0, df=(2,57) → p < 0.001），下一步应做事后两两比较（如 Tukey HSD），确定具体哪两组（或哪几组）之间有显著差异，同时校正多重检验。不能用 3 次独立 t 检验替代——那样会膨胀 I 类错误。

## 总结：统计学在 ML 中的全景

| ML 环节 | 统计工具 | 你的行动 |
|----------|---------|---------|
| **数据探索** | 描述性统计、偏度/峰度 | `.describe()` + 检查分布形状 |
| **特征工程** | Box-Cox 变换（偏度）、Spearman 相关 | 偏态特征 log 变换，找单调关系 |
| **特征选择** | t/卡方/ANOVA + 多重检验校正 | `SelectKBest` + 理解其统计原理 |
| **模型训练** | MLE、MAP（正则化） | 知道损失函数 = 负对数似然 |
| **模型评估** | Bootstrap CI | 不光报告点估计，还要报告区间 |
| **模型比较** | 假设检验 + 效应量 | p 值和 d 一起报告 |
| **实验设计** | 功效分析 → 样本量计算 | A/B 测试开始前确定 n |
| **诊断** | 残差分析、Q-Q、Cook's D | 训练后检查，不是看完 $R^2$ 就完了 |
| **深度思考** | 相关性≠因果性 | SHAP 值 ≠ 因果效应 |
| **哲学框架** | 频率学派 vs 贝叶斯学派 | 理解不确定性、先验知识的利用 |

统计学不是一门孤立的数学课——它是你和数据之间的对话语言，是你从有限样本中提取可信结论的唯一桥梁。掌握它，你的模型将不再是"看起来表现不错"，而是"有证据表明它确实有效"。
