# 写作规范 — EE Tutorial

> 本文档为所有写作者的统一规范。每个章节必须严格遵守。

## 1. 目标读者

**完全零基础小白**。读者可能只有高中数学水平（会解方程、知道坐标系），没有任何电路知识。
- 每个概念必须从"你每天都能见到的东西"开始引入
- 每个公式必须先用文字解释"它在说什么"，再给数学形式
- 不允许出现"显然"、"易得"、"留给读者"这类话
- 所有计算步骤一个都不能跳

## 2. 章节结构模板

每个章节严格按以下结构组织：


## 3. 写作风格

### 3.1 语气
- **对话式**：像在给朋友讲解，不要像写论文
- **鼓励式**："你已经掌握了XX"、"这很简单"
- **提问式**：在引入新概念前先抛出一个问题，引发思考

### 3.2 用词
- **中文为主**：专业术语第一次出现时给出中英文对照，如"电阻（Resistance）"
- **不用口语**：不说"牛X"、"碉堡"等网络用语
- **不用学术腔**：不说"值得注意的是"、"综上所述"

### 3.3 比喻库（必须使用）
| 电学概念 | 推荐比喻 |
|---------|---------|
| 电荷 | 水/水流中的水分子 |
| 电压 | 水压/水位差/重力 |
| 电流 | 水流/车流 |
| 电阻 | 水管粗细/路障 |
| 电容 | 水箱/水库 |
| 电感 | 旋转的水车/惯性 |
| 二极管 | 单向阀 |
| 晶体管 | 水龙头（用小力控制大水流）|
| 放大器 | 扩音器/望远镜 |
| 正弦波 | 海浪/钟摆 |

## 4. 数学公式规范

- 行内公式用 `$...$`：如 $V = IR$
- 独立公式用 `$$...$$`：
```markdown
$$
V = IR
$$
```
- 多行推导用 `\begin{aligned}`：
```markdown
$$
\begin{aligned}
V &= IR \\
  &= 2 \times 5 \\
  &= 10 \text{ V}
\end{aligned}
$$
```
- 每个公式后必须用文字解释其中每个符号的含义
- 数字和单位之间有空格：`10 V`、`5 A`、`3.5 Ω`

## 5. Python 代码规范

- 每个代码块必须有标题注释说明这段代码做什么
- 使用 `numpy`、`scipy`、`matplotlib` 作为主要库
- 代码必须有中文注释
- 每个代码块的预期输出必须写在代码下方
- 绘图代码必须包含标题和轴标签（中文）
- matplotlib 中文字体设置：
```python
plt.rcParams['font.sans-serif'] = ['SimHei', 'Microsoft YaHei', 'Arial Unicode MS']
plt.rcParams['axes.unicode_minus'] = False
```

## 6. 章节长度

- 每章目标 **1200-1800 行**
- 数学基础章节可更长：**1500-2200 行**
- 引言和总结章节可稍短：**1000-1300 行**

## 7. 文件命名

- 章节文件：`01-what-is-ee.md`、`02-charge-voltage-current.md` 等
- 数学基础：`math-physics.md`、`math-complex.md`、`math-signals.md`
- 支持文件：`README.md`、`learning-path.md`

## 8. 前后引用

- 每章开头写明前置章节和下一章
- 引用其他章节时用相对路径：`[欧姆定律](./03-resistance-ohm.md)`
- 引用数学基础时：`[复数与相量](./math-complex.md)`
