## 1. 理想电压源

### 1a. 生活例子：手机充电器上的 5V 标签

翻过你的手机充电器，看上面的小字。你一定会找到一行："Output: 5V ⎓ 2A"。这个"5V"是什么意思？意思是：不管你插的是哪款手机、不管手机正在充电还是已经满了，充电器的 USB 口输出电压**始终保持 5V**（至少它努力保持）。

当然，真实世界没有"始终"这回事。你插上一台快充手机，电流飙到 2A 时，充电器输出电压可能微微掉到 4.9V。但第一次学电路时，我们先把充电器假设成"完美的"：它的输出电压就是 5V，雷打不动。这就是**理想电压源（Ideal Voltage Source）**。

### 1b. 直观理解：水塔里的水面永远在同一个高度

想象你家楼顶有一个巨大的水塔。水塔里有一个超级水泵，无论你用多少水，水泵都能瞬间把水面补回原来的高度。水面高度差（水压）永远不变。你用一滴水，水压是那么多；你把全楼的水龙头都打开，水压还是那么多。

理想电压源就是这个"永不掉水面的水塔"。它两端的电压差是一个固定值 $V_s$，无论你从它那里抽多少电流，这个电压都不变。

### 1c. 形式化定义

**理想电压源（Ideal Voltage Source）**：一种两端元件，其端电压 $v(t)$ 始终等于一个规定值 $V_s$，与流过它的电流 $i(t)$ 的大小和方向无关。

用数学表达：

$$
v(t) = V_s \quad \text{（对任意 } i(t) \text{ 都成立）}
$$

注意一个关键点：电压源**规定了电压**，但**电流由外电路决定**。电源只负责"保持电压不变"，至于电流是多大、往哪流，那是外面负载说了算。

#### V-I 特性曲线

把电压源两端的电压 $V$ 作为纵轴，流过它的电流 $I$ 作为横轴，画出一张图：

```
  V
  │
  ├────────────────────  V = V_s  (一条水平线)
  │
  │
  └────────────────────── I
```

这是一条**水平直线**。不管 $I$ 是多少，$V$ 恒等于 $V_s$。电流可以是 0（开路）、可以是正（正常供电）、可以是负（给电源充电，比如给电池充电），电压始终不动。

#### 电路符号

```
    +  ┌───┐
   ────┤   ├────
       │ V │
   ────┤ s │────
    -  └───┘
```

符号是一个圆圈，里面标着电压值（如 5 V）。长线是正极（+），短线是负极（-）。有时也直接用电池符号（长线 + 短线）代替，但圆圈更通用。

### 1d. 手算示例：电池带一个灯泡

一个理想 9 V 电池给一个 30 Ω 的灯泡供电。求灯泡里的电流和功率。

**步骤 1**：画电路。

```
    +  ┌─────┐
   ────┤ 9V  ├────┬────
       │     │    │
   ────┤     ├──┐ │
    -  └─────┘  │ │
                │ │
             ┌──┘ │
             │    │
             │  ┌─┴─┐
             │  │30Ω│ 灯泡
             │  └─┬─┘
             │    │
             └────┘
```

**步骤 2**：根据欧姆定律，$I = V / R$。

$$
I = \frac{V_s}{R} = \frac{9\text{ V}}{30\ \Omega} = 0.3\text{ A}
$$

**步骤 3**：功率 $P = V \times I$。

$$
P = 9\text{ V} \times 0.3\text{ A} = 2.7\text{ W}
$$

很简单。但这是"理想"情况。下面看真实情况。

#### 实际电池模型：理想源 + 内阻串联

真实的 9 V 电池不是一个纯理想源。电池内部有化学反应，有电极材料，这些东西都会阻碍电流。我们把这种"电池内部的阻碍"叫做**内阻（Internal Resistance）**，记作 $r$ 或 $R_{in}$。

一个实际电池的等效模型是：**一个理想电压源 $V_s$ 串联一个内阻 $r$**。

```
     ┌───┐
  ───┤Vs ├───┬───┐
     │   │   │   │
  ───┤   ├─┐ │   │
     └───┘ │ │   │
           │ │   │
         ┌─┘ │   │
         │   │   │
      ┌──┴──┐│ ┌─┴─┐
      │  r  ││ │R_L│ 负载
      │(内阻)││ │   │
      └──┬──┘│ └─┬─┘
         │   │   │
         └───┘   │
                 │
         ┌───────┘
         │
         └─────────
```

现在重新分析刚才那个灯泡电路。设电池内阻 $r = 2\ \Omega$，负载 $R_L = 30\ \Omega$。

**步骤 1**：$r$ 和 $R_L$ 串联，总电阻 $R_{total} = r + R_L = 2 + 30 = 32\ \Omega$。

**步骤 2**：回路电流 $I = \dfrac{V_s}{R_{total}} = \dfrac{9}{32} = 0.28125\text{ A}$。

**步骤 3**：灯泡实际分到的电压（也就是电池输出端电压）等于 $I \times R_L$：

$$
V_{out} = I \times R_L = 0.28125 \times 30 = 8.4375\text{ V}
$$

不是 9 V！少了 0.5625 V。这少掉的部分去哪了？被内阻 $r$ 吃掉了：

$$
V_r = I \times r = 0.28125 \times 2 = 0.5625\text{ V}
$$

**结论**：当电池有内阻时，输出端电压不是 $V_s$，而是 $V_s$ 减去内阻上的压降。负载越重（$R_L$ 越小，$I$ 越大），内阻压降越大，输出电压越低。

用表格对比三种负载情况（$V_s = 9\text{ V}$，$r = 2\ \Omega$）：

| $R_L$ | 总电阻 $R_{total}$ | 电流 $I$ | 内阻压降 $V_r$ | 输出电压 $V_{out}$ |
|-------|-------------------|----------|---------------|-------------------|
| 100 Ω | 102 Ω | 0.088 A | 0.176 V | 8.824 V |
| 30 Ω | 32 Ω | 0.281 A | 0.563 V | 8.438 V |
| 5 Ω | 7 Ω | 1.286 A | 2.571 V | 6.429 V |

一目了然：负载越重（$R_L$ 越小），电压掉得越厉害。这就是为什么你的手机充电器在快充时会微微发热，而且输出电压不可能完美固定在 5 V。

### 1e. Python 验证：画出理想源 vs 实际源的 V-I 曲线

```python
# ============================================================
# 画出理想电压源和实际电压源的 V-I 特性对比
# ============================================================
import numpy as np
import matplotlib.pyplot as plt

plt.rcParams['font.sans-serif'] = ['SimHei', 'Microsoft YaHei', 'Arial Unicode MS']
plt.rcParams['axes.unicode_minus'] = False

# 参数设置
Vs = 9.0       # 理想电压源的电压 (V)
r = 2.0         # 电池内阻 (Ω)

# 电流范围：从 0 到 2A（模拟从空载到重载）
I = np.linspace(0, 2.0, 100)

# 理想电压源的输出电压：恒等于 Vs（一条水平线）
V_ideal = np.full_like(I, Vs)

# 实际电压源的输出电压：Vs - I * r
V_real = Vs - I * r

# 画图
fig, ax = plt.subplots(1, 1, figsize=(8, 5))
ax.plot(I, V_ideal, 'b-', linewidth=2, label='理想电压源 (V = 9V 不变)')
ax.plot(I, V_real, 'r--', linewidth=2, label=f'实际电池 (Vs=9V, r={r}Ω)')

# 标注关键点
I_load = 0.281  # 接 30Ω 负载时的电流
V_load = Vs - I_load * r
ax.plot(I_load, V_load, 'ro', markersize=8)
ax.annotate(f'接30Ω负载\nI={I_load:.2f}A, V={V_load:.2f}V',
            xy=(I_load, V_load), xytext=(I_load + 0.3, V_load + 0.3),
            arrowprops=dict(arrowstyle='->', color='black'), fontsize=10)

ax.set_xlabel('输出电流 I (A)', fontsize=12)
ax.set_ylabel('输出电压 V (V)', fontsize=12)
ax.set_title('理想电压源 vs 实际电池的 V-I 特性', fontsize=14)
ax.legend(fontsize=11)
ax.grid(True, alpha=0.3)
ax.set_xlim(0, 2.0)
ax.set_ylim(3, 10)

plt.tight_layout()
plt.savefig('ideal-vs-real-voltage-source.png', dpi=150, bbox_inches='tight')
plt.show()

# 打印验证手算结果
print("=" * 50)
print("手算验证：电池内阻模型 (Vs=9V, r=2Ω, RL=30Ω)")
print("=" * 50)
R_L = 30.0
R_total = r + R_L
I_calc = Vs / R_total
V_out = I_calc * R_L
V_r = I_calc * r
print(f"总电阻 R_total = {r} + {R_L} = {R_total} Ω")
print(f"回路电流 I = {Vs} / {R_total} = {I_calc:.4f} A")
print(f"输出电压 V_out = I × R_L = {V_out:.4f} V")
print(f"内阻压降 V_r   = I × r = {V_r:.4f} V")
print(f"验证：V_out + V_r = {V_out + V_r:.4f} V  ≈ Vs = {Vs} V  ✓")
```

运行这段代码，你会看到一条水平线（理想源）和一条向下倾斜的线（实际电池）。两条线在 $I=0$（空载）处重合（都是 9 V），但随着电流增大，实际电池的电压线性下降。这就是内阻模型的核心结论。

### 1f. 常见误区

**误区 1：「电压源提供的电流是自己决定的」**

❌ 电压源只规定电压。电流的大小完全由外部负载决定。同一个 9 V 电池，接 100 Ω 时电流是 0.09 A，接 10 Ω 时电流是 0.9 A。不是电池"想给多少就给多少"，而是负载"要多少就给多少"（直到电池给不起为止）。

✓ 记住口诀：**电压源定电压，电流由负载定**。

**误区 2：「内阻是和电池分开的，是导线上多出来的东西」**

❌ 内阻不是你能摸到的一个独立的电阻元件。它是电池内部所有材料、化学反应、电极结构的综合效果的"等效表示"。你拆开电池看不到一个标着"2 Ω"的小电阻。但我们分析电路时，用一个串联电阻来代表所有这些内部损耗，计算结果和实际测量高度吻合。这就是建模思维。

✓ 内阻是一个**等效参数（Equivalent Parameter）**，不是物理上独立存在的一个电阻。

### 1g. 应用连接

理想电压源模型是你分析一切电路的起点。接下来你会遇到：

- **第5章**（基尔霍夫定律）：用电压源和电阻搭建分压电路

- **真实工程**：每个电源芯片（LDO、DC-DC 转换器）的规格书里都有"输出阻抗"这一项，本质就是内阻。

## 2. 理想电流源

### 2a. 生活例子：恒流充电

有些"快充"不是靠提高电压，而是靠提高电流。比如某些手机快充协议会说"最大 5A 恒流充电"。意思是：不管电池电压是 3.5 V 还是 4.2 V，充电电流一直保持 5 A 不变。这就是理想电流源的思维，它的输出电流是固定的。

另一个更接地气的例子：太阳能电池板。在一定的光照强度下，一块太阳能板输出的电流基本固定。你接一个电阻，电流是那么多；你把两块电阻并联上去，总电流还是差不多那么多（电压下降来补偿）。

### 2b. 直观理解：水塔下面装一个"定流量阀门"

回到水塔的比喻。上一个水塔（电压源）保证了水压不变。现在想象水塔下面有一个特殊的阀门，这个阀门不管水压是高是低，它**每秒通过的水量永远不变**。水塔水位高（水压大），阀门自动关小一点；水塔水位低（水压小），阀门自动开大一点。反正无论如何，流过阀门的水流量（电流）保持恒定。

这就是理想电流源。它"不在乎电压"，只管"电流必须是我规定的那个值"。

注意：和电压源相反，**电流源规定了电流，但电压由外电路决定**。

### 2c. 形式化定义

**理想电流源（Ideal Current Source）**：一种两端元件，其输出电流 $i(t)$ 始终等于一个规定值 $I_s$，与它两端的电压 $v(t)$ 的大小和极性无关。

数学表达：

$$
i(t) = I_s \quad \text{（对任意 } v(t) \text{ 都成立）}
$$

#### V-I 特性曲线

```
  I
  │
  │    I = I_s  (一条垂直线)
  │    │
  │    │
  │    │
  ├────┼──────────────────── V
  │    │
  │    │
```

这是一条**垂直直线**。不管 $V$ 是多少，$I$ 恒等于 $I_s$。电压可以很高（外接大电阻）、可以很低（外接小电阻）、可以是负（外电路给电流源加压），电流始终不动。

#### 电路符号

```
       ┌───┐
  ─────┤Is ├─────
       │   │
  ─────┤   ├─────
       └───┘
```

符号是一个圆圈，里面有一个箭头表示电流方向，旁边标着电流值（如 2 A）。箭头方向代表电流的参考方向。

### 2d. 手算示例：电流源给电阻供电

一个 $I_s = 2\text{ A}$ 的理想电流源连接到一个 $R = 10\ \Omega$ 的电阻上。求电阻两端电压和消耗的功率。

**步骤 1**：画出电路。电流源的箭头方向决定了电流流过电阻的方向。假设电流从上方流入电阻。

```
       ┌─────┐
  ─────┤ 2A  ├────┬────
       │     │    │
  ─────┤     ├──┐ │
       └─────┘  │ │
                │ │
             ┌──┘ │
             │    │
             │  ┌─┴─┐
             │  │10Ω│
             │  └─┬─┘
             │    │
             └────┘
```

**步骤 2**：电流源强制 $I = 2\text{ A}$ 流过电阻。

**步骤 3**：根据欧姆定律，$V = I \times R = 2 \times 10 = 20\text{ V}$。

**步骤 4**：功率 $P = V \times I = 20 \times 2 = 40\text{ W}$。

注意这个计算的方向：是电流决定了电压，不是电压决定电流。电压源是你给电压、算电流；电流源是你给电流、算电压。两者是"对偶"的关系。

#### 实际电流源模型：理想源 + 内阻并联

真实世界没有完美的电流源。一个实际电流源内部也有损耗。它的模型是：**一个理想电流源 $I_s$ 并联一个内阻 $R_{in}$**。

```
       ┌───┐
  ─────┤Is ├─────┬──────
       │   │     │
  ─────┤   ├──┐  │
       └───┘  │  │
              │  │
           ┌──┴──┐│
           │ Rin ││  R_L (负载)
           │     ││
           └──┬──┘│
              │   │
              └───┘
```

当外接负载 $R_L$ 时，$I_s$ 产生的电流会分两路：一路流过内阻 $R_{in}$，一路流过负载 $R_L$。负载电流不再是 $I_s$，而是：

$$
I_L = I_s \times \frac{R_{in}}{R_{in} + R_L}
$$

这是**分流公式（Current Divider）**，我们在第5章会详细推导。这里先混个脸熟。

举个例子：$I_s = 1\text{ A}$，$R_{in} = 1000\ \Omega$，$R_L = 100\ \Omega$。

$$
I_L = 1 \times \frac{1000}{1000 + 100} = 1 \times \frac{1000}{1100} \approx 0.909\text{ A}
$$

理想情况下负载应该拿到 1 A，实际只拿到 0.909 A，约 9% 的电流被内阻"分流"走了。内阻越大（越接近理想开路），损耗越小。这和电压源正好相反：电压源内阻越小越好，电流源内阻越大越好。

### 2e. Python 验证：电流源 V-I 特性

```python
# ============================================================
# 画出理想电流源和实际电流源的 I-V 特性对比
# ============================================================
import numpy as np
import matplotlib.pyplot as plt

plt.rcParams['font.sans-serif'] = ['SimHei', 'Microsoft YaHei', 'Arial Unicode MS']
plt.rcParams['axes.unicode_minus'] = False

Is = 2.0        # 理想电流源的输出电流 (A)
Rin = 50.0      # 实际电流源的内阻 (Ω)

# 电压范围
V = np.linspace(0, 100, 200)

# 理想电流源：电流恒等于 Is
I_ideal = np.full_like(V, Is)

# 实际电流源：I = Is - V/Rin（电压越高，内阻分走的电流越多）
I_real = Is - V / Rin
I_real = np.clip(I_real, 0, Is)  # 电流不能为负

fig, ax = plt.subplots(1, 1, figsize=(8, 5))
ax.plot(V, I_ideal, 'b-', linewidth=2, label='理想电流源 (I = 2A 不变)')
ax.plot(V, I_real, 'r--', linewidth=2, label=f'实际电流源 (Is=2A, Rin={Rin}Ω)')

# 标注负载点：接 10Ω 电阻时，V = Is * (Rin // R_L) 近似
R_L = 10.0
R_par = (Rin * R_L) / (Rin + R_L)
V_load = Is * R_par
I_load = V_load / R_L
ax.plot(V_load, I_load, 'ro', markersize=8)
ax.annotate(f'接10Ω负载\nV={V_load:.1f}V, I_RL={I_load:.2f}A',
            xy=(V_load, I_load), xytext=(V_load + 10, I_load + 0.1),
            arrowprops=dict(arrowstyle='->', color='black'), fontsize=10)

ax.set_xlabel('输出电压 V (V)', fontsize=12)
ax.set_ylabel('输出电流 I (A)', fontsize=12)
ax.set_title('理想电流源 vs 实际电流源 I-V 特性', fontsize=14)
ax.legend(fontsize=11)
ax.grid(True, alpha=0.3)

plt.tight_layout()
plt.savefig('ideal-vs-real-current-source.png', dpi=150, bbox_inches='tight')
plt.show()

# 手算验证
print("=" * 50)
print("手算验证：实际电流源 (Is=2A, Rin=50Ω, RL=10Ω)")
print("=" * 50)
I_L_calc = Is * Rin / (Rin + R_L)
V_L_calc = I_L_calc * R_L
print(f"并联等效电阻 R_par = (50×10)/(50+10) = {R_par:.2f} Ω")
print(f"输出电压 V = Is × R_par = {V_load:.2f} V")
print(f"负载电流 I_L = {Is} × 50/(50+10) = {I_L_calc:.4f} A")
print(f"内阻分流 I_Rin = V / Rin = {V_load/Rin:.4f} A")
print(f"验证：I_L + I_Rin = {I_L_calc + V_load/Rin:.4f} A ≈ Is = {Is} A  ✓")
```

### 2f. 常见误区

**误区 1：「电流源输出端不能开路」**

❌ 这是初学者最危险的一个误区（从安全角度）。理想电流源强制输出固定电流，如果你把输出端开路（不接任何负载），电流总要找一条路走。理想电流源会把输出电压无限升高，试图在断开的空气里"打出火花"来维持电流，结果就是**设备损坏**。真实电流源一般有保护电路，但分析中必须铭记：**电流源忌讳开路**。这和电压源忌讳短路是一对。

✓ 电压源忌讳短路（电流无穷大），电流源忌讳开路（电压无穷大）。

**误区 2：「电流源和电压源完全对立，选哪个都一样」**

❌ 不对立，而是**对偶（Dual）**。很多电路中两种源都会出现，而且你还可以把一个实际电压源（理想源 + 串联内阻）等效变换为一个实际电流源（理想源 + 并联内阻）。这叫**电源等效变换（Source Transformation）**，是一个非常有用的分析技巧，第5章会详细讲。

### 2g. 应用连接

电流源在实际电路中以各种形态出现：

- 太阳能电池板：光照不变时输出电流基本恒定
- BJT 晶体管（将在第15章讲到）：集电极电流近似为电流源，受基极电流控制
- LED 驱动电路：LED 需要恒流驱动，而不是恒压驱动
- 电池充电器：很多锂电池充电芯片是以恒流方式工作的

## 3. 受控源

### 3a. 生活例子：你拧水龙头就是在"受控"

打开你家厨房的水龙头。你的手拧动龙头把手的角度（输入），控制了水流的大小（输出）。手拧的角度本身不是水，但它的变化让水量跟着变。这就是"受控"的核心思想：**一个量控制另一个量**。

在电路中，这种"受控"关系同样存在。一个元件输出的电压或电流，可以被**另一个地方的电压或电流**控制。这类元件叫做**受控源（Dependent Source）**。

### 3b. 直观理解：用一个小信号推一个大信号

为什么需要受控源？因为你经常需要做一件这样的事：用一个微弱的信号（比如麦克风输出的几毫伏电信号）去控制一个强大的输出（比如推动喇叭发出震耳欲聋的声音）。如果直接用麦克风的微弱信号去推喇叭，喇叭根本不会响。你需要一个"中间人"：它感知到输入信号的电压/电流的大小，然后按比例放大后输出。

这个"中间人"在电路模型中就是受控源。最常见的一类受控源就是**晶体管（Transistor）** 的小信号模型：基极和发射极之间的一点点电压变化，能控制集电极和发射极之间流过几十倍甚至几百倍的电流。这就是"用小力气拧阀门，让大水冲出来"。

### 3c. 形式化定义

**受控源（Dependent Source）**：一种四端元件（或可简化为四端），其输出电压或电流不是固定值，而是由电路中另一处的电压或电流按照一定比例关系决定。

受控源有四种，按"谁控制谁"来分类：

| 简写 | 全称 | 输入（控制量） | 输出（被控量） | 增益参数 |
|------|------|--------------|--------------|---------|
| **VCVS** | 电压控制电压源 (Voltage-Controlled Voltage Source) | 另一处的电压 $v_x$ | 电压 $v_o = \mu \cdot v_x$ | $\mu$（无量纲，电压增益） |
| **VCCS** | 电压控制电流源 (Voltage-Controlled Current Source) | 另一处的电压 $v_x$ | 电流 $i_o = g_m \cdot v_x$ | $g_m$（西门子 S，跨导） |
| **CCVS** | 电流控制电压源 (Current-Controlled Voltage Source) | 另一处的电流 $i_x$ | 电压 $v_o = r_m \cdot i_x$ | $r_m$（欧姆 Ω，跨阻） |
| **CCCS** | 电流控制电流源 (Current-Controlled Current Source) | 另一处的电流 $i_x$ | 电流 $i_o = \beta \cdot i_x$ | $\beta$（无量纲，电流增益） |

#### 电路符号：菱形

受控源的符号不是圆形，而是**菱形（◇）**。一看这个形状就知道"这是一个受控源，不是独立源"。

```
VCVS:                    VCCS:
      + ┌───┐                  ┌───┐
  ──────┤ ◇ ├──────       ─────┤ ◇ ├──────
        │   │                  │   │
  ──────┤   ├──────       ─────┤   ├──────
      - └───┘                  └───┘
  v_o = μ·v_x              i_o = g_m·v_x

CCVS:                    CCCS:
      + ┌───┐                  ┌───┐
  ──────┤ ◇ ├──────       ─────┤ ◇ ├──────
        │   │                  │   │
  ──────┤   ├──────       ─────┤   ├──────
      - └───┘                  └───┘
  v_o = r_m·i_x            i_o = β·i_x
```

菱形里的"+"和"-"表示输出电压的参考极性，箭头表示输出电流的参考方向。

### 3d. 手算示例：含 VCVS 的简单电路

电路如下：一个独立电压源 $V_s = 10\text{ V}$ 给一个 5 Ω 电阻供电，电阻上的电压作为控制电压 $v_x$。这个 $v_x$ 输入到一个 VCVS，增益为 $\mu = 3$，VCVS 的输出接一个 10 Ω 的负载电阻 $R_L$。求负载上的电压和电流。

```
     控制侧                     输出侧
  ┌──────────────┐        ┌──────────────┐
  │              │        │              │
  │   +  ┌──┐   │   v_x  │   +  ┌──┐   │
  └──┬───┤Vs├───┬─────────┼──────┤◇ ├───┬──┐
     │   │10V│  │         │      │μ=3│  │  │
     │   └──┘   │         │      └──┘  │  │
     │          │         │            │  │
     │        ┌─┴─┐       │          ┌─┴─┐│
     │        │5Ω │ v_x   │          │10Ω││ R_L
     │        └─┬─┘       │          └─┬─┘│
     │          │         │            │  │
     └──────────┘         └────────────┘  │
                                      ┌───┘
                                      │
                                      └──── 公共地
```

**步骤 1**：控制侧分析。$v_x$ 就是 5 Ω 电阻两端的电压。由于电压源 $V_s = 10\text{ V}$ 直接加在 5 Ω 两端（假设导线理想），所以：

$$
v_x = 10\text{ V}
$$

**步骤 2**：VCVS 输出电压 $v_o = \mu \cdot v_x = 3 \times 10 = 30\text{ V}$。

**步骤 3**：负载电流 $I_L = \dfrac{v_o}{R_L} = \dfrac{30}{10} = 3\text{ A}$。

**步骤 4**：负载功率 $P_L = v_o \times I_L = 30 \times 3 = 90\text{ W}$。

注意这个电路的妙处：输入侧只提供了 10 V 的电压，功耗很小（$P_{in} \approx 10^2 / 5 = 20\text{ W}$），但输出侧通过 VCVS 给出了 30 V、90 W 的功率给负载。**多出来的能量不是"凭空变出来的"，是 VCVS 这个元件本身需要外接电源供电（图中没画）。** 受控源是一个"放大器"，它把电源提供的直流能量，按照输入信号的比例，"搬运"到输出端。

再做一个 CCCS 的例子。一个独立电流源 $I_s = 0.1\text{ A}$ 流过控制支路，控制电流 $i_x = 0.1\text{ A}$。CCCS 增益 $\beta = 50$，输出接一个 4 Ω 的负载 $R_L$。

**步骤 1**：$i_x = I_s = 0.1\text{ A}$。

**步骤 2**：CCCS 输出电流 $i_o = \beta \cdot i_x = 50 \times 0.1 = 5\text{ A}$。

**步骤 3**：负载电压 $V_L = i_o \times R_L = 5 \times 4 = 20\text{ V}$。

**步骤 4**：负载功率 $P_L = V_L \times i_o = 20 \times 5 = 100\text{ W}$。

### 3f. 常见误区

**误区 1：「受控源和独立源一样，不需要外部供电」**

❌ 独立源（电池、发电机）自己产生能量。受控源是一个**模型**，它不代表一个能自己产生能量的东西。受控源输出给负载的能量，实际上来自于电路中未画出的"偏置电源"。比如一个晶体管放大器电路，受控源模型的输出功率来自直流电源 $V_{CC}$，不是凭空产生的。

✓ 受控源模型是用来**描述晶体管等器件的行为**的数学工具，它本身不是能量来源。真正的能量来自外部供电。

**误区 2：「四种受控源太难记了，学三个就够了」**

❌ 每一种在实际中都有对应。VCVS 对应运算放大器（Op-Amp），VCCS 对应场效应管（FET），CCCS 对应双极型晶体管（BJT），CCVS 在传感器接口电路中常出现。你在后续章节会一个接一个遇到它们。

✓ 现在的任务是先认全四种，知道它们的增益参数叫什么。后面的章节会一个个深入。

### 3g. 应用连接

受控源是整个模拟电路分析的"语言"。你以后读到的每一个晶体管放大器电路，第一步都是把晶体管替换成它的受控源小信号模型，然后用线性电路的方法（欧姆定律 + 基尔霍夫定律）来分析。

- **运算放大器（Op-Amp）**，第 17-18 章：可以建模为一个极高增益的 VCVS
- **BJT 晶体管**，第 15 章：小信号模型核心是一个 CCCS
- **MOSFET 晶体管**，第 16 章：小信号模型核心是一个 VCCS
- **变压器模型**：可以建模为受控源组合

## 4. 开关

### 4a. 生活例子：按一下灯就亮，再按一下灯就灭

这是你每天都在用的东西。墙上的电灯开关、手机上的电源键、笔记本的合盖传感器。它的功能简单到不能再简单：让电路接通或断开。

### 4b. 直观理解：水阀

开关就是一个"水阀门"。阀门打开，水从水管流过；阀门关上，水被截断。电路开关同理：**闭合（Closed）** 时像一段理想导线（短路），电流畅通无阻；**断开（Open）** 时像被剪断的导线（开路），电流为零。

### 4c. 形式化定义

**理想开关（Ideal Switch）**：一种两端元件，只有两种状态。

| 状态 | 电路表现 | 等效模型 | 电阻 |
|------|---------|---------|------|
| **闭合 (Closed / ON)** | 两端之间如同导线 | 短路 (Short Circuit) | $R = 0\ \Omega$ |
| **断开 (Open / OFF)** | 两端之间完全不通 | 开路 (Open Circuit) | $R = \infty\ \Omega$ |

电路符号：

```
闭合状态:  ────────        断开状态:  ────╱  ╲────
         (导线连通)                  (杠杆翘起)
```

实际开关有**接触电阻（Contact Resistance）**，通常是几十毫欧（$m\Omega$）级别。在低压大电流场景下，这个小小的接触电阻会产生可观的发热和压降。但在大多数日常电路分析中，我们把它当成理想开关来处理。

### 4d. 开关通断对电路的影响：一个简单得不能再简单的例子

电路：一个 5 V 电池串联一个开关和一个 50 Ω 的灯泡。

**开关闭合时**：

$$
I = \frac{5\text{ V}}{50\ \Omega} = 0.1\text{ A}
$$

灯亮，功率 $P = 5 \times 0.1 = 0.5\text{ W}$。

**开关断开时**：

整个回路断了，电流 $I = 0$。灯泡不亮。电池两端电压仍然是 5 V，但它没有向外输出电流。

就这么简单。但这个简单到极点的元件，是所有数字电路的基础。数字逻辑里的"0"和"1"，本质上就是晶体管的"断开"和"闭合"（导通）。几十亿个这样的"开关"在芯片里以纳秒级的速度快速通断，就实现了你看到的一切计算。

### 4e. 常见误区

**误区 1：「开关断开时，开关两端电压也是 0」**

❌ 开关断开时，开关两端的电压等于电源电压（5 V）。因为开关断开的那一瞬间，整个回路的电流为零，电阻上没有压降，电源的全部电压都"堆"在了开关的断开处。用万用表量断开开关的两端，你会量到 5 V。

✓ 断开时电流为零（$I=0$），但电压等于电源电压（$V=V_s$），不是零。

**误区 2：「开关就是一个简单的机械部件，没什么好学的」**

❌ 别小看开关。在实际工程中，开关的"抖动（Bounce）"是一个经典问题：机械开关在接触瞬间不是干脆利落地从断开变成闭合，而是会快速地"弹跳"几次（几个毫秒内通断好几次），导致数字电路误判为多次按键。去抖动（Debouncing）是嵌入式系统中每个工程师都会遇到的问题。

## 5. 电路拓扑

### 5a. 生活例子：地铁线路图

打开你手机里的地铁线路图 App。那上面画的是什么？不是真实的地理距离，不是每个拐弯的弧度，而是**站点和站点之间的连接关系**。你从"西直门"到"国贸"，要经过哪几个站、在哪换乘，这和地面的实际距离几乎无关，只和"谁连着谁"有关。

电路拓扑（Circuit Topology）研究的就是这个：**元件之间"谁连着谁"，而不关心元件本身是什么、导线有多长、形状如何。** 这就像地铁线路图一样，只关心连接关系，不关心布局。

### 5b. 直观理解：元件是站点，导线是线路

把每个电路元件想象成地铁站，导线是地铁线路。两个元件之间的连接线，不管画成直线还是弯弯曲曲，只要它们是同一根导线，电气上就是同一个点（因为理想导线假设，电压处处相等）。

拓扑分析就做一件事：**数清楚这张"地铁线路图"上有几个站点、几段线路、几个闭合的环。**

### 5c. 形式化定义

四个核心概念，一个一个来。

#### 节点（Node）

**节点**：两个或更多元件的连接点。在理想导线假设下，一根导线上任意一点的电压是相同的，所以**一整根导线（以及所有被它连在一起的点）算作一个节点**。

```
   ────┬────┬────  这三个 T 形交叉点都在同一根导线上，
       │    │      电压相同。它们属于**同一个节点**。
       │    │
```

关键规则：**不是"交叉点才算节点"**。一个元件的两个端点本身也是节点。比如一个电阻有两个端子，每个端子所在的导线各算一个节点。

#### 支路（Branch）

**支路**：连接在两个节点之间的一段路径，上面有一个或多个元件串联。最简单的支路就是"一个元件连在两个节点之间"。如果多个元件串联在两个节点之间，这一段整体算一条支路。

```
   节点A ──[R1]── 节点B ──[R2]── 节点C
```

这里有两条支路：支路 1 是 [R1] 连接节点 A 和 B，支路 2 是 [R2] 连接节点 B 和 C。

#### 回路（Loop）

**回路**：电路中的一个闭合路径。从某个节点出发，沿着支路走，不重复走同一段，最终回到起点。一个电路中通常有很多个回路。

#### 网孔（Mesh）

**网孔**：一种特殊的回路，**在这个回路内部不包含任何其他支路**。可以理解为"最小的、不能继续被分割的回路"。

注意：网孔只定义在**平面电路（Planar Circuit）**上。非平面电路没有网孔的概念（或者说不能用网孔法分析）。

#### 平面电路 vs 非平面电路

**平面电路（Planar Circuit）**：可以在一个平面上画出，且任意两条导线除了在节点处相交外，不产生其他交叉。

```
平面电路：
  ┌──R1──┬──R3──┐
  │      │      │
  +      R2     R4
  Vs     │      │
  -      │      │
  │      │      │
  └──────┴──────┘
(所有交叉都在节点处，没有"跳线")

非平面电路：
     A ──── B
    ╱ ╲    ╱ ╲
   ╱   ╲  ╱   ╲
  C───── D ──── E
   ╲   ╱  ╲   ╱
    ╲ ╱    ╲ ╱
     F ──── G
(无论怎么画，总有一些导线要交叉而不能全在节点处)
```

经典的非平面电路例子是**惠斯通电桥**不是一个非平面电路（它可以画成平面的）。真正的非平面电路比较少见，你在初学阶段碰到的绝大多数电路都是平面电路。

### 5d. 手算示例：数节点、支路、回路、网孔

**例题 1**：最基础的串联电路

```
   +  ┌───┐
  ────┤Vs ├────[R1]────[R2]────
      │   │                    │
  ────┤   ├────────────────────┘
   -  └───┘
```

- **节点数**：2 个。导线把 Vs 的负极、R2 的右端连在一起，这是一个节点（记为 N1）。Vs 的正极和 R1 的左端连在一起，这是第二个节点（N2）。R1 的右端和 R2 的左端也连在一起，但它们和 N1、N2 都没有导线直连。等等，让我们仔细数。

- R1 的右端和 R2 的左端通过导线连接：节点 B
- R2 的右端和 Vs 的下端（-）通过导线连接：节点 C

所以有 **3 个节点** A、B、C。

- **支路数**：3 条。支路 1 = Vs（节点A到C），支路 2 = R1（节点A到B），支路 3 = R2（节点B到C）。
- **回路数**：1 个。A → R1 → B → R2 → C → Vs → A。
- **网孔数**：1 个。因为只有一个回路，它本身也是网孔。

### 5e. Python 验证：用 NetworkX 分析电路拓扑

```python
# ============================================================
# 用图论工具分析电路拓扑：节点、支路、回路
# ============================================================
import numpy as np
import matplotlib.pyplot as plt

plt.rcParams['font.sans-serif'] = ['SimHei', 'Microsoft YaHei', 'Arial Unicode MS']
plt.rcParams['axes.unicode_minus'] = False

# ==========================================
# 分析一个桥式电路的拓扑
# ==========================================
#   电路：4节点(A,B,C,D)、6条支路(Vs,R1,R2,R3,R4,R5)
#   支路连接关系：
#     Vs: A ↔ D
#     R1: A ↔ B
#     R2: A ↔ C
#     R3: B ↔ D
#     R4: C ↔ D
#     R5: B ↔ C

branches = [
    ('Vs', 'A', 'D'),
    ('R1', 'A', 'B'),
    ('R2', 'A', 'C'),
    ('R3', 'B', 'D'),
    ('R4', 'C', 'D'),
    ('R5', 'B', 'C'),
]

nodes = sorted(set([n for _, a, b in branches for n in (a, b)]))
n = len(nodes)
b = len(branches)

print("=" * 50)
print("拓扑分析：桥式电路")
print("=" * 50)
print(f"节点集合: {nodes}")
print(f"节点数 n = {n}")
print(f"支路数 b = {b}")
print(f"独立回路数 l = b - n + 1 = {b} - {n} + 1 = {b - n + 1}")
print()

# 列出所有支路
print("支路列表：")
for name, a, b_node in branches:
    print(f"  {name}: {a} ↔ {b_node}")
print()

# 构建邻接表
adj = {node: [] for node in nodes}
for name, a, b_node in branches:
    adj[a].append((b_node, name))
    adj[b_node].append((a, name))

print("邻接关系（每个节点连着哪些支路）：")
for node in nodes:
    conns = [f"{name}(→{other})" for other, name in adj[node]]
    print(f"  节点 {node}: {', '.join(conns)}")
print()

# 手动列出网孔（平面电路中，网孔是最小闭合回路）
meshes = [
    ['R1', 'R5', 'R2'],   # A→B→C→A
    ['R3', 'R4', 'R5'],   # B→D→C→B
    ['Vs', 'R3', 'R1'],   # A→B→D→A (经过 Vs)
]

print("网孔（最小闭合回路）：")
for i, m in enumerate(meshes, 1):
    print(f"  网孔 M{i}: {' → '.join(m)}")
print()

# 验证：每个网孔确实是闭合的
def is_closed(mesh, branches):
    # 统计每个节点在网孔支路中出现的次数
    node_count = {}
    for name in mesh:
        for _, a, b in branches:
            if _ == name:
                node_count[a] = node_count.get(a, 0) + 1
                node_count[b] = node_count.get(b, 0) + 1
    # 闭合回路的充要条件：每个节点出现偶数次（通常2次，进出各一）
    # 但网孔中每个节点恰好连接两条支路
    return all(v == 2 for v in node_count.values())

for i, m in enumerate(meshes, 1):
    closed = is_closed(m, branches)
    print(f"  网孔 M{i} 闭合验证: {'✓' if closed else '✗'}")

# 可视化电路拓扑图
fig, ax = plt.subplots(1, 1, figsize=(6, 6))

# 手动定义节点位置（模拟电路布局）
pos = {
    'A': (0, 1),
    'B': (1, 2),
    'C': (1, 0),
    'D': (2, 1),
}

# 画支路（导线）
edge_style = {
    'Vs': ('black', 2.5, 'solid'),
    'R1': ('red', 2, 'solid'),
    'R2': ('blue', 2, 'solid'),
    'R3': ('green', 2, 'solid'),
    'R4': ('orange', 2, 'solid'),
    'R5': ('purple', 2, 'dashed'),
}

for name, a, b_node in branches:
    color, lw, ls = edge_style[name]
    x1, y1 = pos[a]
    x2, y2 = pos[b_node]
    ax.plot([x1, x2], [y1, y2], color=color, linewidth=lw, linestyle=ls)

    # 在支路中间标注元件名
    mx, my = (x1 + x2) / 2, (y1 + y2) / 2
    offset_x = 0.05 * (y2 - y1)
    offset_y = -0.05 * (x2 - x1)
    ax.text(mx + offset_x, my + offset_y, name, fontsize=10,
            ha='center', va='center',
            bbox=dict(boxstyle='round,pad=0.2', facecolor='white', alpha=0.8))

# 画节点
for node, (x, y) in pos.items():
    ax.plot(x, y, 'ko', markersize=12)
    ax.text(x, y, node, fontsize=12, ha='center', va='center', color='white', fontweight='bold')

ax.set_xlim(-0.5, 2.8)
ax.set_ylim(-0.5, 2.8)
ax.set_aspect('equal')
ax.axis('off')
ax.set_title('桥式电路拓扑图\n(节点=4, 支路=6, 独立回路=网孔=3)', fontsize=13)

plt.tight_layout()
plt.savefig('bridge-circuit-topology.png', dpi=150, bbox_inches='tight')
plt.show()
```

### 5f. 常见误区

**误区 1：「一根导线就是一个节点，所以导线越多节点越多」**

❌ 一整根连续导线（包括所有分支）算作**一个**节点，因为在理想导线假设下，这根导线上所有点的电压完全相同。你用手指沿着导线追，只要没有经过任何元件，你触碰到的所有点都属于同一个节点。

✓ 判断方法：从一个元件端子出发，沿着导线走，直到碰到下一个元件为止。这条路径上的所有点都属于同一个节点。

**误区 2：「网孔就是回路，回路就是网孔」**

❌ 网孔是回路的一种，是"最小的、内部不含其他支路的回路"。一个电路可能有很多回路（理论上每一条闭合路径都算），但网孔只有那几个"基本格子"。比如桥式电路有 3 个网孔，但可能的回路组合远多于 3 个。

✓ 关系：所有网孔都是回路，但不是所有回路都是网孔。网孔 ⊂ 回路。

**误区 3：「非平面电路画不出来，所以不存在」**

❌ 非平面电路虽然不常见，但确实存在。比如某些射频电路、多层 PCB 布线，在二维平面上画不出不交叉的版本。不过对于本教程的绝大部分内容，你面对的都是平面电路，所以网孔分析完全够用。

### 5g. 应用连接

拓扑分析是后面两大"杀器"的基础：

- **网孔电流法（Mesh Current Method）**，第 6 章：你定义每个网孔里有一个"假想的循环电流"，然后对每个网孔写 KVL 方程。网孔数 = 方程数。
- **节点电压法（Node Voltage Method）**，第 6 章：你选一个参考节点（通常是地），然后对每个非参考节点写 KCL 方程。节点数 - 1 = 方程数。

在工程实践中，SPICE 仿真软件（你以后会用到）就是基于这些拓扑分析自动建立方程并求解的。你画好电路图，软件自动识别节点、支路、回路，然后解方程。你学拓扑，就是为了理解这背后的原理。

## 6. 端口与网络

### 6a. 生活例子：你根本不知道手机充电器里面有什么

你每天把充电器插到手机上。你不知道充电器里面是变压器还是开关电源，不知道用了哪些芯片，不知道电路板长什么样。但你不需要知道。你只需要知道：**这边两个脚插墙上的 220 V，那边一个 USB 口输出 5 V，最大 2 A。** 对你来说，充电器就是一个"黑盒子"，只关心它对外表现出的特性。

这就是端口网络思维的精髓。

### 6b. 直观理解：水的"进水口"和"出水口"

你家的自来水系统对你来说也是一个"端口"。墙上有一个水龙头（出水口），下水道有一个排水口。你不知道水从水厂到你家经过了几个泵站、几公里管道。你只知道：打开水龙头，水流出来，用过的水从下水道排走。

一端口网络（也称为单端口网络或二端网络）就是：**两个接线端子，一个进、一个出。你只关心这两个端子上的电压和电流，不管里面有什么。**

### 6c. 形式化定义

**一端口网络（One-Port Network）**：只有两个对外接线端子的电路。外部只能通过这两个端子对它施加电压或抽取电流。

```
        ┌──────────┐
  I →   │          │
  ──────┤  一端口  ├──────
  +     │  网络    │
  V     │          │
  -     │          │
  ──────┤          ├──────
        └──────────┘
```

一端口网络的最简单例子：一个电阻。它两个端子，你从两端看进去，只看到一个阻值 $R$。一个电池也是一个一端口网络，从两端看进去，你看到一个电压源串联内阻。

**二端口网络（Two-Port Network）**：有四个对外接线端子，分成两对：输入端口（Port 1）和输出端口（Port 2）。

```
         I1 →              I2 →
  ────────┬──────────────────┬──────
  +       │                  │      +
  V1      │   二端口网络     │     V2
  -       │                  │      -
  ────────┼──────────────────┼──────
          │                  │
       输入端口            输出端口
```

二端口网络的经典例子是放大器：输入端口接一个微弱信号 $(V_1, I_1)$，输出端口给出放大后的信号 $(V_2, I_2)$。你不需要知道放大器里面有多少个晶体管、多少个电阻，你只需要知道它的"二端口参数"：输入阻抗、输出阻抗、电压增益、电流增益。这四个参数（或它们的变形）完全描述了从端口看进去的所有特性。

**黑盒思维**是工程中最强大的思维方式之一。当你面对一个复杂系统时，你把它分成若干个"黑盒子"，每个黑盒子的内部细节你暂时不关心，你只关心它对外表现出的端口特性。然后你把黑盒子们连起来，就搭出了一个大系统。这道理在写代码（函数封装）、修汽车（发动机是一个黑盒子）、造房子（混凝土是一个黑盒子）中都一样。

### 6d. 常见误区

**误区 1：「端口就是两个随便的接线点」**

❌ 端口有一对隐含的要求：流入一个端子的电流必须等于流出另一个端子的电流。这在单端口网络中自然满足（电流进出只有两个端子），但在多端口网络中如果不满足，就不能称为"端口"。

✓ 端口的定义要求：$I_{in} = I_{out}$，即流入端口一端子的电流必须等于从另一端子流出的电流。

**误区 2：「只有复杂的集成电路才用端口模型」**

❌ 任何东西都可以看作一个端口网络。一个电阻是一端口，一个扬声器是一端口，一根天线是一端口，甚至一根导线也可以看作一端口（电阻为 0 的端口）。端口思维和复杂度无关，它只是一种观察电路的角度：你选择只看这两个端子上的 V 和 I，而有意忽略内部细节。

## 7. 常见误区（全章汇总）

除了前面各节已经提到的误区，这里再补充几个跨节常犯的错误：

**误区 1：「独立源和受控源可以互换使用」**

❌ 完全不同。独立源的值是固定的（$V_s$ 或 $I_s$），不依赖电路中其他量。受控源的值是变化的，随控制量的变化而变化。在列电路方程时，独立源是"已知常数"，受控源是"含未知量的项"。

**误区 2：「节点就是画图时导线的交叉点」**

❌ 节点和画图方式无关。一根连续导线上的所有点都是同一个节点，即使导线上没有画任何"交叉"。反过来，两条导线交叉但画了跳线弧（没有小黑点），它们不是同一个节点，因为电气上没有连通。

**误区 3：「数节点和支路不重要，解方程能解就行」**

❌ 不对。知道 $b - n + 1$ 这个公式，你就能在动手解方程之前就知道需要几个独立方程。这个数字是电路复杂度的直接度量。一个有经验的分析者看电路图，第一件事就是数节点和独立回路，评估问题的规模。

**题 8 解答**：

这是一个经典的黑盒等效问题。设黑盒内部是一个实际电压源模型：理想源 $V_s$ 串联内阻 $R_s$。

第一次测量（$R_{L1} = 10\ \Omega$）：

$$
I_1 = \frac{V_s}{R_s + 10}, \quad V_1 = I_1 \times 10 = \frac{10V_s}{R_s + 10} = 8\text{ V}
$$

第二次测量（$R_{L2} = 5\ \Omega$）：

$$
I_2 = \frac{V_s}{R_s + 5}, \quad V_2 = I_2 \times 5 = \frac{5V_s}{R_s + 5} = 6\text{ V}
$$

从两个方程解出 $V_s$ 和 $R_s$。

由第一个方程：$10V_s = 8(R_s + 10) \Rightarrow 10V_s = 8R_s + 80 \quad (1)$

由第二个方程：$5V_s = 6(R_s + 5) \Rightarrow 5V_s = 6R_s + 30 \quad (2)$

将 (2) 乘以 2：$10V_s = 12R_s + 60 \quad (3)$

(1) = (3)：$8R_s + 80 = 12R_s + 60$

$80 - 60 = 12R_s - 8R_s \Rightarrow 20 = 4R_s \Rightarrow R_s = 5\ \Omega$

代入 (2)：$5V_s = 6 \times 5 + 30 = 60 \Rightarrow V_s = 12\text{ V}$

**答案**：黑盒内部是一个理想 12 V 电压源串联一个 5 Ω 内阻。

验证：接 10 Ω 时，$I = 12/(5+10) = 0.8\text{ A}$，$V = 0.8 \times 10 = 8\text{ V}$ ✓。接 5 Ω 时，$I = 12/(5+5) = 1.2\text{ A}$，$V = 1.2 \times 5 = 6\text{ V}$ ✓。
